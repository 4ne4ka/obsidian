1. **Разделяй и властвуй (Divide and Conquer):** Если классов много, их нужно разбивать на **модули**.
2. **Инкапсуляция:** Сложность системы должна быть скрыта за простыми **интерфейсами**. Лектор приводит аналогию с автомобилем: чтобы управлять им, не нужно знать устройство двигателя, достаточно интерфейса в виде руля и педалей.

- Разрабатывать программу **последовательно**, создавая и проверяя рабочие прототипы один за другим.
- **Декомпозировать** общую задачу на более мелкие сущности, скрывая детали реализации.

**Модульность и её ограничения**

Первый шаг к упрощению — **модульная структура**. Классы объединяются в модули, которые можно разрабатывать отдельно. Это дает преимущества:
- **Возможность замены

Однако, простого разбиения на папки (модульности) часто **недостаточно**. Модульность сама по себе не описывает такие **взаимосвязи и правила** между модулями.

**Иерархия и два ключевых механизма**

В разработке иерархия может проявляться в двух формах:
1. Построение сложных **абстракций** на основе более простых.
2. **Декомпозиция** сложной задачи на более простые подзадачи (что очень хорошо реализуется в парадигме ООП).

Для построения иерархии в Java используются два основных механизма:
1. **Агрегация и Композиция:** Создание новых классов путем **включения** объектов других классов в качестве полей.
2. **Наследование:** Создание новых классов на основе существующих путем **добавления** новых методов или **изменения** (переопределения) реализации существующих.

**Отличие Агрегации от Композиции (с примером из C++)**

- **Ключевое отличие:** **Время жизни** включенного объекта.
- **Агрегация:** Объект создается **вне** класса и передается в него (например, через конструктор). Время жизни этого объекта **не зависит** от времени жизни содержащего его объекта.
    - _Пример на C++:_ В класс `PointPrinter` передается указатель на внешний объект класса `Point`. Уничтожение `PointPrinter` не повлечет за собой уничтожение `Point`

- **Композиция:** Объект создается **внутри** класса. Его время жизни **полностью совпадает** с временем жизни содержащего его объекта.
    - _Пример на C++:_ Объект класса `Point` создается внутри класса `PointPrinter` как член данных по значению (а не по указателю). Уничтожение `PointPrinter` приведет к уничтожению и его внутреннего объекта `Point`.

**Реализация Агрегации и Композиции в Java**

1. **Создание базового класса `Point`:**
    - Создается класс `Point` с полями для координат (например, `x` и `y`).
    - Для инициализации полей добавляются конструкторы: как конструктор с параметрами, так и конструктор по умолчанию (без параметров).
2. **Реализация Агрегации:**
    - Создается новый класс (например, `PointPrinterAggregation`), цель которого — добавить функциональность печати для точки.
    - Внутри этого класса объявляется **ссылка** на объект типа `Point` (напр., `Point p;`). Лектор подчеркивает ключевое отличие Java от C++: это объявление **не выделяет память** для объекта `Point`, а лишь создает ссылку, которая изначально может указывать в `null`.
    - Чтобы реализовать агрегацию, в класс `PointPrinterAggregation` добавляется конструктор, который **принимает объект `Point` извне** и сохраняет его в свою внутреннюю ссылку. Время жизни этого переданного объекта `Point` **не зависит** от времени жизни объекта `PointPrinterAggregation`.
    - Добавляется метод `print()` (или `printLn()`), который, используя переданную ссылку, печатает координаты точки (через `System.out.println`).
~~~Java
Point myPoint = new Point(10, 20); // Создали точку
PointPrinterAggregation printer = new PointPrinterAggregation(myPoint); // Передали ее в агрегирующий класс
printer.print(); // Печатаем
// Объект 'myPoint' продолжит существовать, даже если 'printer' будет уничтожен
~~~

**Реализация Композиции:**

- Создается другой класс (например, `PointPrinterComposition`).
- Внутри этого класса объект `Point` создается **не как ссылка, а непосредственно как поле** (напр., `Point p = new Point();`). Объект `Point` создается **внутри** конструктора класса `PointPrinterComposition` или при объявлении поля.
- Чтобы задать координаты точки, можно добавить в `PointPrinterComposition` конструктор, который принимает параметры `x` и `y` и передает их внутреннему объекту `Point`, либо добавить методы-геттеры/сеттеры для взаимодействия с ним.
- Ключевой момент: время жизни объекта `Point` **полностью совпадает** с временем жизни содержащего его объекта `PointPrinterComposition`. Они создаются и уничтожаются вместе.
~~~Java
PointPrinterComposition compPrinter = new PointPrinterComposition(5, 15); // Создаем объект композиции
compPrinter.print(); // Печатаем. Внутренняя точка создалась вместе с 'compPrinter'
// Уничтожение 'compPrinter' повлечет за собой уничтожение и его внутренней точки.
~~~


**Важные концепции: Перегрузка и Переопределение**
**Перегрузка (Overloading):**
- Это возможность иметь несколько методов с **одним и тем же именем**, но с **разными наборами параметров** (разный тип или количество).
- Реализация, которая будет вызвана, зависит от того, какие аргументы передаются при вызове.

**Переопределение (Overriding):**
- Это когда производный класс объявляет метод с **таким же именем и сигнатурой** (типом и количеством параметров), как и у метода в базовом классе.
- -Если метод **не виртуальный**, то вызываемая реализация зависит от типа **ссылки** (или указателя) на объект.
- Если метод **виртуальный**, то вызываемая реализация зависит от типа **самого объекта**, который был создан, а не от типа ссылки на него.

**Особенности наследования в Java**
**Синтаксис и доступ:**
- В Java **нет спецификатора доступа** (как `public`, `protected`, `private` в C++) при объявлении наследования.
- Ключевое слово для наследования — **`extends`**.
~~~Java
class B extends A { // Класс B наследуется от класса A
    // ...
}
~~~

**Виртуальные методы и переопределение:**
-  Java **все методы являются "виртуальными"** по умолчанию. Это означает, что поведение, продемонстрированное в примере на C++ с `virtual`, в Java работает всегда.

- Если нужно **запретить** переопределение метода в классах-потомках, метод помечается ключевым словом **`final`**.
~~~Java
 class A {
    public final void cannotOverride() { // Этот метод нельзя переопределить
        System.out.println("Это финальный метод");
    }
}
~~~

**Множественное наследование:**
    - В Java **отсутствует множественное наследование классов** (один класс не может наследоваться от двух других классов).
    - Однако, эта проблема решается с помощью **интерфейсов**, которые позволяют реализовывать множественное "наследование" поведения.

- Для явного указания того, что метод переопределяет метод родительского класса, используется аннотация **`@Override`**.  
- Это **рекомендуется** делать, так как аннотация помогает компилятору найти ошибки (например, опечатку в имени метода или несовпадение сигнатуры).

 
 **Создается базовый класс `A`:**
~~~Java
    class A {
        public void f1() {
            System.out.println("A.f1() called");
        }
        public void f2() {
            System.out.println("A.f2() called");
        }
    }
~~~
**Создается класс-наследник `B`:**
~~~Java
    class B extends A {
        @Override       // Аннотация, указывающая на переопределение
        public void f2() { // Этот метод переопределяет A.f2()
            System.out.println("B.f2() called");
        }
        // Метод f1() не переопределен, поэтому будет унаследован от класса A
    }
~~~
**Создается метод для тестирования:**  
    Лектор создает метод (например, `callMethods`), который принимает объект типа `A` и вызывает у него `f1()` и `f2()`. Это демонстрирует полиморфизм: хотя ссылка имеет тип `A`, для объекта типа `B` будет вызван переопределенный метод `B.f2()`.

**Доступ к функциональности базового класса с помощью `super`**
- Конструктор базового класса **по умолчанию** (без параметров) вызывается **автоматически** из конструктора потомка.
- Чтобы вызвать **конкретный конструктор** базового класса (не по умолчанию), используется ключевое слово **`super`** с параметрами. Этот вызов **должен быть первым оператором** в конструкторе потомка (за исключением новых правил в Java 25).
~~~Java
class A {
    public A() {
        System.out.println("A's default constructor");
    }
    public A(int value) {
        System.out.println("A's constructor with value: " + value);
    }
}

class B extends A {
    public B() {
        // Здесь автоматически вызовется super() - конструктор A по умолчанию
        System.out.println("B's default constructor");
    }
    public B(int value) {
        super(value); // Явный вызов конструктора A(int value)
        System.out.println("B's constructor with value: " + value);
    }
}
~~~
- Чтобы вызвать метод из базового класса, который был переопределен в текущем классе, также используется ключевое слово **`super`**.
~~~Java
class B extends A {
    @Override
    public void f2() {
        super.f2(); // Сначала вызываем оригинальный метод из класса A
        System.out.println("B.f2() added functionality"); // Потом добавляем свое
    }
}
~~~



**Создание и реализация интерфейса в Java**
- В Java для этого есть специальное ключевое слово **`interface`** (в отличие от C++, где для этого использовались абстрактные классы).
- В интерфейсе объявляются методы **без реализации** (как чисто виртуальные методы в C++). Ключевое слово `public` писать необязательно, все методы интерфейса по умолчанию public.
~~~Java
interface ICalc { // Имя интерфейса часто начинают с 'I'
    void init();       // Метод для инициализации (например, обнуления)
    int calc(int a, int b); // Метод для выполнения операции
}
~~~

- Для реализации интерфейса используется ключевое слово **`implements`** (а не `extends`).
- Класс, реализующий интерфейс, **обязан переопределить (override)** все его методы.
~~~Java
class Sum implements ICalc {
    private int value; // Внутреннее состояние

    @Override
    public void init() {
        value = 0; // Инициализируем начальное значение
    }

    @Override
    public int calc(int a, int b) {
        value = a + b; // Выполняем операцию
        return value;
    }
}
~~~


**Использование интерфейса для создания универсального алгоритма**
**Создаем класс-алгоритм:**  
Создадим класс `Processor`, который будет применять операцию `ICalc` ко всем элементам массива. Он не знает, какая именно это операция (сложение, умножение), он знает только интерфейс.
~~~Java
class Processor {
    private ICalc op; // Агрегация: ссылка на интерфейс

    public Processor(ICalc operation) { // Конструктор принимает реализацию интерфейса
        this.op = operation;
    }

    public int run(int[] array) {
        op.init(); // Инициализируем операцию
        int val = 0;
        for (int i : array) { // Цикл for-each по всем элементам массива
            val = op.calc(val, i); // Применяем операцию к текущему результату и элементу
        }
        return val; // Возвращаем итоговый результат
    }
}
~~~
**Собираем все вместе:**  
Теперь мы можем использовать один и тот же алгоритм (`Processor`) с разными реализациями интерфейса (`ICalc`).
~~~Java
public static void main(String[] args) {
    int[] data = {1, 2, 3, 4, 5}; // Создаем массив данных

    // Вариант 1: Используем реализацию Sum (сложение)
    ICalc sumOp = new Sum();
    Processor sumProcessor = new Processor(sumOp);
    int resultSum = sumProcessor.run(data);
    System.out.println("Sum: " + resultSum); // Выведет: Sum: 15

    // Вариант 2: Создаем другую реализацию - умножение
    class Mult implements ICalc {
        private int value;
        @Override
        public void init() { value = 1; } // Для умножения начинаем с 1!
        @Override
        public int calc(int a, int b) { value = a * b; return value; }
    }

    ICalc multOp = new Mult();
    Processor multProcessor = new Processor(multOp);
    int resultMult = multProcessor.run(data);
    System.out.println("Multiplication: " + resultMult); // Выведет: Multiplication: 120
}
~~~

**Анонимные классы**
- Позволяют реализовать интерфейс или наследовать класс прямо в месте создания объекта.
- Синтаксис: `new InterfaceName() { /*реализация методов*/ }`
~~~Java
ICalc printOp = new ICalc() { // Анонимный класс
    @Override
    public void init() { System.out.print("Array: "); }
    @Override
    public int calc(int a, int b) {
        System.out.print(b + " "); // Печатаем элемент массива
        return 0; // Возвращаемое значение не важно для печати
    }
};

Processor printProcessor = new Processor(printOp);
printProcessor.run(data); // Выведет: Array: 1 2 3 4 5
~~~
