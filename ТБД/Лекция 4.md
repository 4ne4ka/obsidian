# Конспект: SQL — Вложенные запросы (Subqueries)

## 1. Определение и назначение

**Вложенный запрос (подзапрос, subquery)** — это запрос SQL, расположенный внутри другого запроса (основного или внешнего запроса).

*   **Принцип работы:** Подзапрос выполняется **один раз** перед выполнением основного запроса. Его результат используется в условии (`WHERE`, `HAVING`) или в списке `SELECT` основного запроса.
*   **Синтаксис:**
    ```sql
    SELECT select_list
    FROM table
    WHERE expr operator (SELECT select_list FROM table);
    ```

### 1.1. Что может возвращать подзапрос?
Подзапрос может возвращать разные типы результатов:
1.  **Таблицу** (набор строк и столбцов) — используется в `FROM` (производная таблица) или с операторами `IN`, `EXISTS`.
2.  **Строку** (набор столбцов) — используется в `WHERE` для сравнения с составным условием.
3.  **Столбец** (один столбец, много строк) — используется с операторами `IN`, `ANY`, `ALL`.
4.  **Одно значение** (скаляр) — используется в `WHERE` с операторами сравнения (`=`, `>`, `<` и т.д.) или в списке `SELECT`.


## 2. Типы подзапросов по возвращаемому значению

| Тип подзапроса | Возвращаемое значение | Операторы сравнения |
| :--- | :--- | :--- |
| **Скалярный (Single-row)** | Одна строка с одним столбцом (одно значение). | `=`, `>`, `<`, `>=`, `<=`, `<>` |
| **Многострочный (Multi-row)** | Несколько строк с одним столбцом. | `IN`, `ANY`, `ALL` |
| **Многостолбцовый (Multi-column)** | Одна строка с несколькими столбцами или несколько строк с несколькими столбцами. | `IN` (для кортежей), `EXISTS` |

### 2.1. Скалярный подзапрос (Single-row Subquery)
Возвращает ровно одно значение. Должен быть гарантированно однострочным (например, с использованием агрегатной функции без `GROUP BY` или с условием на уникальный ключ).

*   **Пример:** Найти сотрудников, зарабатывающих больше, чем сотрудник с кодом 9812.
    ```sql
    SELECT ename, sal
    FROM emp
    WHERE sal > (SELECT sal FROM emp WHERE empno = 9812);
    ```

### 2.2. Многострочные подзапросы (Multi-row Subqueries)
Возвращают несколько строк. Используются со специальными операторами.

| Оператор | Значение |
| :--- | :--- |
| **`IN`** | Равно любому значению из списка (эквивалентно `= ANY`). |
| **`ANY`** | Сравнение со значением, которое "существует". Возвращает TRUE, если условие выполняется хотя бы для одного значения из набора. |
| **`ALL`** | Сравнение со значением, которое "для всех". Возвращает TRUE, если условие выполняется для всех значений из набора. |

*   **Пример с `ANY`:** Найти сотрудников, у которых зарплата меньше, чем у **любого** (хотя бы одного) клерка, но которые сами не являются клерками.
    ```sql
    SELECT empno, ename, job
    FROM emp
    WHERE sal < ANY (SELECT sal FROM emp WHERE job = 'CLERK')
      AND job <> 'CLERK';
    ```

*   **Пример с `ALL`:** Найти сотрудников, чья зарплата больше, чем **все** средние зарплаты по отделам (т.е. больше максимальной из средних).
    ```sql
    SELECT empno, ename, job
    FROM emp
    WHERE sal > ALL (SELECT AVG(sal) FROM emp GROUP BY deptno);
    ```

### 2.3. Многостолбцовые подзапросы (Multi-column Subqueries)
Позволяют сравнивать несколько столбцов одновременно, используя кортежи.

*   **Пример:** Найти сотрудников, у которых пара (зарплата, комиссионные) совпадает с парой любого сотрудника из отдела 30. (Функция `NVL` используется для замены `NULL` на -1, чтобы избежать проблем с сравнением).
    ```sql
    SELECT ename, deptno, sal, comm
    FROM emp
    WHERE (sal, NVL(comm, -1)) IN
          (SELECT sal, NVL(comm, -1)
           FROM emp
           WHERE deptno = 30);
    ```

## 3. Правила и рекомендации по использованию подзапросов

1.  **Скобки:** Подзапросы всегда заключаются в круглые скобки `()`.
2.  **Порядок выполнения:** Сначала выполняется внутренний (самый глубокий) подзапрос, затем внешний.
3.  **Сравнение типов:** Тип данных, возвращаемый подзапросом, должен быть совместим с типом данных в левой части оператора сравнения.
4.  **`ORDER BY`:** В подзапросе (внутри `WHERE` или `HAVING`) предложение `ORDER BY` обычно не имеет смысла и не допускается (за исключением подзапросов в `FROM` или с `ROW_NUMBER()`). Сортируется только внешний запрос.
5.  **Однострочные операторы и многострочный результат:** Использование операторов `=`, `>`, `<` с подзапросом, который возвращает более одной строки, вызовет ошибку.

## 4. Места использования подзапросов

### 4.1. Подзапрос в `WHERE` (наиболее частый случай)
Используется для фильтрации строк основной таблицы на основе данных из другой таблицы.

### 4.2. Подзапрос в списке `SELECT` (скалярный подзапрос)
Позволяет подтянуть вычисляемое значение для каждой строки результата. Должен возвращать ровно одно значение.

*   **Пример:** Для каждого абонента вывести его имя и общую сумму длительности звонков.
    ```sql
    SELECT ks,
           (SELECT sfn || ' ' || sln FROM s WHERE s.ks = f.ks) AS sname,
           SUM(dur)
    FROM f
    GROUP BY ks
    ORDER BY ks;
    ```

### 4.3. Подзапрос в `FROM` (производная таблица, inline view)
Результат подзапроса используется как временная таблица для внешнего запроса.

*   **Пример:** Вычислить суммарную длительность звонков для каждого абонента, а затем объединить с таблицей абонентов.
    ```sql
    SELECT s.ks, s.sfn, s.sln, sm.sdur
    FROM s
    JOIN (SELECT ks, SUM(dur) AS sdur FROM f GROUP BY ks) AS sm
      ON sm.ks = s.ks
    ORDER BY s.ks;
    ```

### 4.4. Подзапрос в `HAVING`
Используется для фильтрации групп на основе агрегированных данных из другой таблицы.


## 5. Альтернативы подзапросам: `WITH` (CTE) и `JOIN`

В презентации демонстрируются различные способы решения одной и той же задачи (вывод абонентов с суммой звонков):

1.  **Подзапрос в списке `SELECT`:**
    ```sql
    SELECT ks, sfn, sln,
           (SELECT SUM(dur) FROM f WHERE f.ks = s.ks) AS sum
    FROM s;
    ```

2.  **`WITH` (Common Table Expression — CTE):**
    Создает временное именованное выражение, которое можно использовать в основном запросе. Улучшает читаемость сложных запросов.
    ```sql
    WITH sm(ks, sdur) AS (
        SELECT ks, SUM(dur) FROM f GROUP BY ks
    )
    SELECT s.ks, s.sfn, s.sln, sm.sdur
    FROM s
    JOIN sm ON sm.ks = s.ks
    ORDER BY s.ks;
    ```

3.  **`JOIN` с группировкой:**
    Наиболее прямой и часто самый эффективный способ.
    ```sql
    SELECT s.ks AS sks, sfn, sln, SUM(dur) AS sdur
    FROM s
    JOIN f ON s.ks = f.ks
    GROUP BY s.ks, sfn, sln
    ORDER BY s.ks;
    ```

## 6. Ключевые выводы

1.  **Подзапросы** — мощный инструмент для создания гибких запросов, позволяющий использовать данные из одной таблицы для фильтрации или вычислений в другой.
2.  Важно различать **скалярные** (одно значение) и **многострочные** подзапросы, так как для них используются разные операторы сравнения (`=` vs `IN`/`ANY`/`ALL`).
3.  **Операторы `ANY` и `ALL`** позволяют сравнивать значение с набором значений, полученных из подзапроса.
4.  **Многостолбцовые подзапросы** полезны, когда условие фильтрации требует одновременного совпадения нескольких атрибутов.
5.  Для сложных запросов стоит рассмотреть альтернативы:
    *   **`JOIN`** часто более читаем и эффективен для связывания таблиц.
    *   **`WITH` (CTE)** улучшает читаемость, разбивая сложную логику на шаги.
6.  **Порядок выполнения:** Внутренний подзапрос выполняется первым, его результат подставляется во внешний запрос.
