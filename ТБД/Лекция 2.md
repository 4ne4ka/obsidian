# Конспект: SQL — Соединение таблиц (JOINs)

## 1. Основы соединения таблиц

### 1.1. Проблема
Данные, необходимые для ответа на запрос, часто хранятся в нескольких таблицах. Чтобы извлечь эти данные в одном результирующем наборе, необходимо соединить таблицы.

### 1.2. Декартово произведение (Cross Join)
Если в предложении `FROM` перечислить несколько таблиц без условия соединения в `WHERE`, результатом будет декартово произведение. Каждая строка из первой таблицы соединяется с каждой строкой из второй таблицы.
*   **Результат:** Количество строк = (строк в таблице A) * (строк в таблице B). Это почти всегда бессмысленный и ресурсоемкий результат.

### 1.3. Реализация соединения (Классический синтаксис)
Соединение используется для извлечения данных из двух или более таблиц. Условия соединения определяются в предложении `WHERE`.
*   **Квалификация имен (Префиксы):** Если в соединяемых таблицах есть столбцы с одинаковыми именами, необходимо использовать префикс (имя таблицы или псевдоним), чтобы указать, из какой таблицы брать столбец.

```sql
SELECT table1.column, table2.column
FROM table1, table2
WHERE table1.column1 = table2.column2;
```

## 2. Типы соединений (Классический синтаксис)

### 2.1. Экви-соединение (Equi-Join)
Соединяет таблицы на основе равенства значений в столбцах. В большинстве случаев это соединение первичного и внешнего ключей.

*   **Пример:** Вывести имя сотрудника и название его отдела.
    ```sql
    SELECT e.ename, d.dname
    FROM emp e, dept d
    WHERE e.deptno = d.deptno;
    ```

### 2.2. Некви-соединение (Non-Equi-Join)
Соединяет таблицы на основе оператора, отличного от равенства (`=`) — например, `BETWEEN`, `>`, `<`, `>=`, `<=`.

*   **Пример:** Определить грейд (уровень) зарплаты сотрудника из таблицы `salgrade`.
    ```sql
    SELECT e.ename, e.sal, s.grade
    FROM emp e, salgrade s
    WHERE e.sal BETWEEN s.losal AND s.hisal;
    ```

### 2.3. Внешнее соединение (Outer Join) — Классический синтаксис (Oracle)
Используется для включения в результат строк, которые не удовлетворяют условию соединения. В классическом синтаксисе Oracle это обозначается оператором `(+)`.

*   **`(+)` с правой стороны:** `LEFT OUTER JOIN` (все строки из левой таблицы).
*   **`(+)` с левой стороны:** `RIGHT OUTER JOIN` (все строки из правой таблицы).

*   **Пример:** Вывести все отделы и сотрудников, которые в них работают. Если в отделе нет сотрудников, он все равно будет показан (со значением `NULL` для данных сотрудника).
    ```sql
    SELECT e.ename, d.deptno, d.dname
    FROM emp e, dept d
    WHERE e.deptno(+) = d.deptno   -- (+) означает "добавить NULL, если нет совпадения в emp"
    ORDER BY d.deptno;
    ```

### 2.4. Само-соединение (Self-Join)
Соединение таблицы с самой собой. Это полезно, когда в таблице есть иерархические связи (например, сотрудник и его менеджер, который также является сотрудником). Для этого необходимо использовать псевдонимы.

*   **Пример:** Найти имя сотрудника и имя его менеджера.
    ```sql
    SELECT worker.ename AS "Employee", manager.ename AS "Manager"
    FROM emp worker, emp manager
    WHERE worker.mgr = manager.empno;
    ```

## 3. Современный синтаксис JOIN (ANSI SQL 92 и новее)

Этот синтаксис является предпочтительным, так как он отделяет условия соединения (`ON`) от условий фильтрации (`WHERE`), делая запрос более читаемым и менее подверженным ошибкам.

### 3.1. `INNER JOIN` (Внутреннее соединение)
Возвращает только те строки, для которых найдено соответствие в обеих таблицах. Эквивалентно классическому `WHERE table1.col = table2.col`.

```sql
SELECT *
FROM TableA
INNER JOIN TableB ON TableA.name = TableB.name;
```

### 3.2. `LEFT OUTER JOIN` (Левое внешнее соединение)
Возвращает все строки из левой таблицы (`TableA`) и соответствующие им строки из правой таблицы (`TableB`). Если соответствия нет, то для столбцов правой таблицы будут возвращены `NULL`.

```sql
SELECT *
FROM TableA
LEFT OUTER JOIN TableB ON TableA.name = TableB.name;
```

### 3.3. `RIGHT OUTER JOIN` (Правое внешнее соединение)
Возвращает все строки из правой таблицы (`TableB`) и соответствующие им строки из левой таблицы (`TableA`). Если соответствия нет, то для столбцов левой таблицы будут возвращены `NULL`.

### 3.4. `FULL OUTER JOIN` (Полное внешнее соединение)
Возвращает все строки из обеих таблиц. Если для строки из одной таблицы нет соответствия в другой, для столбцов другой таблицы будут `NULL`.

```sql
SELECT *
FROM TableA
FULL OUTER JOIN TableB ON TableA.name = TableB.name;
```

## 4. Расширенные сценарии и исключения

### 4.1. Поиск несоответствующих записей (Anti-Join)
Часто требуется найти строки в одной таблице, у которых нет соответствующих строк в другой. Это достигается с помощью внешнего соединения и фильтрации `NULL` в предложении `WHERE`.

*   **Левое анти-соединение (Anti Left Join):** Найти записи в `TableA`, у которых нет соответствия в `TableB`.
    ```sql
    SELECT *
    FROM TableA
    LEFT OUTER JOIN TableB ON TableA.name = TableB.name
    WHERE TableB.id IS NULL;  -- Фильтр оставляет только несоответствующие строки из A
    ```

*   **Полное анти-соединение (Full Outer Anti-Join):** Найти записи в обеих таблицах, у которых нет соответствия в другой.
    ```sql
    SELECT *
    FROM TableA
    FULL OUTER JOIN TableB ON TableA.name = TableB.name
    WHERE TableA.id IS NULL OR TableB.id IS NULL;
    ```


## 5. Сравнение синтаксиса: Oracle Classic vs ANSI SQL

| Тип соединения | ANSI SQL (Современный стандарт) | Классический синтаксис Oracle |
| :--- | :--- | :--- |
| **INNER JOIN** | `FROM tab1 INNER JOIN tab2 ON col1 = col2` | `FROM tab1, tab2 WHERE col1 = col2` |
| **LEFT JOIN** | `FROM tab1 LEFT JOIN tab2 ON col1 = col2` | `FROM tab1, tab2 WHERE col1 = col2(+)` |
| **RIGHT JOIN** | `FROM tab1 RIGHT JOIN tab2 ON col1 = col2` | `FROM tab1, tab2 WHERE col1(+) = col2` |
| **FULL JOIN** | `FROM tab1 FULL JOIN tab2 ON col1 = col2` | *(Не поддерживается в классическом синтаксисе Oracle)* |

## 6. Ключевые выводы и правила

1.  **Современный синтаксис (`INNER JOIN`, `LEFT JOIN` и т.д.) является предпочтительным.** Он делает код более понятным, отделяя логику соединения от логики фильтрации.
2.  **Всегда используйте псевдонимы** для таблиц, особенно при соединении трех и более таблиц или при само-соединениях. Это сокращает код и повышает его читаемость.
3.  **Всегда указывайте условие соединения.** Пропуск условия `ON` (или условия в `WHERE` для классического синтаксиса) приведет к декартову произведению, которое может быть очень медленным и потреблять много ресурсов.
4.  **Для иерархических данных** (например, сотрудник-менеджер) используйте **само-соединение (Self-Join)**.
5.  **Для поиска "сиротских" записей** (записей без связи) используйте **внешнее соединение** с проверкой `IS NULL`.
