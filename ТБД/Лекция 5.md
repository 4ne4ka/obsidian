# Конспект: Хранимые процедуры и PL/SQL (PostgreSQL)

## 1. Эволюция архитектуры приложений

Выделяют три эпохи архитектуры приложений:
1.  **Мэйнфреймы (1960–70-е):** «Толстый» сервер, «тонкие» терминалы.
2.  **Персональные компьютеры (1980-е):** Локальная обработка данных на клиенте.
3.  **Веб-эра (1990-е – настоящее время):** Трехуровневая архитектура.

---

## 2. Модели клиент-серверного взаимодействия

### 2.1. Двухуровневая модель (Two-Tier)
Приложение (клиент) напрямую взаимодействует с сервером БД.
*   **Плюс:** Простота.
*   **Минус:** Высокая нагрузка на клиента, проблемы с масштабированием.

### 2.2. Трехуровневая модель (Three-Tier)
Между клиентом и сервером БД находится промежуточный слой — **сервер приложений** (Application Server).
*   **Задачи сервера приложений:** Кэширование, балансировка нагрузки, бизнес-логика.
*   **«Тонкий клиент»:** Клиенту достаточно браузера.

### 2.3. Серверное программирование (Server-Side Coding)
Вместо передачи данных на клиент для обработки, часть логики выносится непосредственно на сервер БД в виде:
*   **Хранимых процедур (Stored Procedures)** — вызываются явно.
*   **Триггеров (Triggers)** — выполняются автоматически при изменении данных.

**Преимущества:**
*   Снижение сетевого трафика.
*   Переиспользование кода (одна процедура используется разными клиентами).
*   Повышение безопасности.

## 3. Доступ к SQL из языков программирования

Для взаимодействия приложений с СУБД используются API:

| API | Язык | Описание |
| :--- | :--- | :--- |
| **JDBC** | Java | Java Database Connectivity. Стандарт для Java. |
| **ODBC** | C, C++, C#, VB | Open Database Connectivity. Низкоуровневый API. |
| **ADO.NET** | C#, .NET | Надстройка над ODBC для .NET платформы. |
| **Embedded SQL** | C, C++, COBOL | Встраивание SQL-запросов непосредственно в код. |

### 3.1. Основные шаги работы с БД из приложения
1.  **Открыть соединение (Open Connection)** — подключиться к серверу.
2.  **Выполнить операции (Operations)** — отправить запросы, получить результаты.
3.  **Закрыть соединение (Close Connection)** — освободить ресурсы.

### 3.2. JDBC (краткий алгоритм)
1.  Импорт библиотеки `java.sql.*`.
2.  Загрузка драйвера: `Class.forName("driver")`.
3.  Создание объекта соединения: `getConnection()`.
4.  Создание объекта `Statement` (или `PreparedStatement`).
5.  Выполнение запроса: `executeQuery()`.
6.  Обработка результата в `ResultSet` (двумерная таблица).

## 4. Встраиваемый SQL (Embedded SQL)

SQL-запросы встраиваются в код языка высокого уровня (C, C++, Java, COBOL), который называется **host language**.

*   **Синтаксис:** `EXEC SQL <оператор SQL>;`
*   **Обмен данными:** Переменные host-языка используются в SQL с префиксом `:` (двоеточие).

### 4.1. Курсоры в Embedded SQL
Курсор — это итератор, позволяющий обрабатывать результат запроса по одной строке.

```c
EXEC SQL BEGIN DECLARE SECTION;
    int credit_amount;
    char si[20], sn[20];
EXEC SQL END DECLARE SECTION;

// Объявление курсора
EXEC SQL declare c cursor for
    SELECT ID, name FROM student WHERE tot_cred > :credit_amount;

// Открытие курсора (выполнение запроса)
EXEC SQL open c;

// Получение строки
EXEC SQL fetch c into :si, :sn;

// Закрытие курсора
EXEC SQL close c;
```

### 4.2. Динамический SQL
Позволяет выполнять SQL-команды, сформированные во время выполнения (например, введенные пользователем).

```c
EXEC SQL BEGIN DECLARE SECTION;
    varchar sqlupdatestring[256];
EXEC SQL END DECLARE SECTION;

// Подготовка (PREPARE) и выполнение (EXECUTE)
EXEC SQL PREPARE sqlcommand FROM :sqlupdatestring;
EXEC SQL EXECUTE sqlcommand;
```

## 5. PL/pgSQL — Язык процедурного программирования PostgreSQL

PL/pgSQL — это полностью портируемый, высокопроизводительный язык для написания хранимых процедур и функций. Он включает любые SQL-команды и процедурные конструкции (условия, циклы).

### 5.1. Объекты PL/pgSQL

| Объект | Описание | Вызов |
| :--- | :--- | :--- |
| **Процедура (PROCEDURE)** | Модуль, который не возвращает значение. Может изменять данные. | `CALL имя();` |
| **Функция (FUNCTION)** | Модуль, который возвращает значение (скаляр, таблица). | `SELECT имя();` |
| **Триггер (TRIGGER)** | Функция, вызываемая автоматически при событии (INSERT, UPDATE, DELETE). | Автоматически. |
| **Пакет / Схема (PACKAGE)** | Объединяет несколько процедур и функций в один логический блок. | — |

---

## 6. Синтаксис PL/pgSQL (PostgreSQL)

### 6.1. Создание процедуры

```sql
CREATE [OR REPLACE] PROCEDURE имя([режим] имя_аргумента тип)
LANGUAGE 'plpgsql'
AS $BODY$
DECLARE
    -- объявление переменных
BEGIN
    -- тело процедуры
END;
$BODY$;
```

*   **Режимы аргументов:**
    *   `IN` — входной параметр (по умолчанию).
    *   `OUT` — выходной параметр (возвращает значение через параметр).
    *   `INOUT` — входной и выходной одновременно.

### 6.2. Создание функции

```sql
CREATE [OR REPLACE] FUNCTION имя([режим] имя_аргумента тип)
RETURNS тип_результата
LANGUAGE 'plpgsql'
AS $BODY$
BEGIN
    -- тело функции
    RETURN выражение;
END;
$BODY$;
```

### 6.3. Объявление переменных

```sql
DECLARE
    имя [CONSTANT] тип [NOT NULL] [DEFAULT | := | = выражение];
```

*   **CONSTANT** — значение нельзя изменить после инициализации.
*   **NOT NULL** — запрещает присвоение `NULL`.
*   **DEFAULT** — значение по умолчанию.

**Примеры:**
```sql
DECLARE
    quantity integer DEFAULT 32;
    url varchar := 'http://site.com';
    user_id CONSTANT integer := 10;
```

### 6.4. Присваивание

```sql
переменная := выражение;
-- или
переменная = выражение;
```

### 6.5. Условные операторы

```sql
-- IF-THEN
IF условие THEN
    действия;
END IF;

-- IF-THEN-ELSE
IF условие THEN
    действия1;
ELSE
    действия2;
END IF;

-- IF-THEN-ELSIF-ELSE
IF условие1 THEN
    действия1;
ELSIF условие2 THEN
    действия2;
ELSE
    действия3;
END IF;

-- CASE
CASE выражение
    WHEN значение1 THEN действия1;
    WHEN значение2 THEN действия2;
    ELSE действия3;
END CASE;
```

### 6.6. Циклы

#### Безусловный цикл (LOOP)
```sql
[<<label>>] LOOP
    -- действия
    EXIT [label] [WHEN условие]; -- выход из цикла
    CONTINUE [label] [WHEN условие]; -- переход на следующую итерацию
END LOOP [label];
```

#### Цикл с параметром (FOR по диапазону)
```sql
FOR имя IN [REVERSE] начало .. конец [BY шаг] LOOP
    -- действия
END LOOP;
```
*   `REVERSE` — обратный порядок.
*   `BY` — шаг итерации.

#### Цикл по результатам запроса (FOR по курсору)
```sql
FOR целевая_переменная IN запрос LOOP
    -- действия
END LOOP;
```

## 7. Курсоры в PL/pgSQL

Курсор инкапсулирует запрос и позволяет читать результат по строкам.

### 7.1. Объявление курсора
```sql
DECLARE
    имя_курсора CURSOR [(параметры)] FOR SELECT-запрос;
```

### 7.2. Операции с курсором
| Операция | Команда | Описание |
| :--- | :--- | :--- |
| **Открытие** | `OPEN имя_курсора;` | Выполняет запрос и готовит результат. |
| **Чтение строки** | `FETCH [направление] FROM имя_курсора INTO переменная;` | Получает следующую строку. |
| **Перемещение** | `MOVE [направление] FROM имя_курсора;` | Перемещает позицию без чтения данных. |
| **Обновление/Удаление** | `UPDATE ... WHERE CURRENT OF имя_курсора;` | Изменяет/удаляет текущую строку курсора (требуется `FOR UPDATE` в запросе). |
| **Закрытие** | `CLOSE имя_курсора;` | Освобождает ресурсы. |

### 7.3. Направления FETCH
*   `NEXT` — следующая (по умолчанию).
*   `PRIOR` — предыдущая.
*   `FIRST` — первая.
*   `LAST` — последняя.
*   `ABSOLUTE n` — строка по абсолютному номеру.
*   `RELATIVE n` — строка относительно текущей.

## 8. Разбор практических примеров из терминала

### 8.1. Процедура `test_progression()`
Вычисляет сумму чисел от 1 до 100 с помощью цикла `FOR`.

```sql
CREATE OR REPLACE PROCEDURE public.test_progression()
LANGUAGE 'plpgsql'
AS $BODY$
DECLARE
  s1 NUMERIC := 0;
  ii NUMERIC;
BEGIN
  FOR ii IN 1..100 LOOP
    s1 := s1 + ii;
  END LOOP;
  RAISE NOTICE 'Сумма прогрессии равна %', s1;
END;
$BODY$;
```

*   **Вызов:** `CALL test_progression();`
*   **Результат:** `ЗАМЕЧАНИЕ: Сумма прогрессии равна 5050`

---

### 8.2. Процедура `calculate_total_salary_1()`
Использует **неявный курсор** (цикл `FOR ... IN SELECT`) для суммирования всех зарплат.

```sql
CREATE OR REPLACE PROCEDURE public.calculate_total_salary_1()
LANGUAGE 'plpgsql'
AS $BODY$
DECLARE
  total_salary   NUMERIC := 0.00;
  current_record RECORD;
BEGIN
  FOR current_record IN SELECT * FROM lecturers LOOP
    total_salary := total_salary + current_record.salary;
  END LOOP;
  RAISE NOTICE 'Полученная сумма = %', total_salary;
END;
$BODY$;
```

*   **Вызов:** `CALL calculate_total_salary_1();`
*   **Результат:** `ЗАМЕЧАНИЕ: Полученная сумма = 7135.56`

---

### 8.3. Процедура `calculate_total_salary_2()`
Использует **явный курсор** с условием:
*   Если зарплата `>= 1000` — добавляет к сумме.
*   Если зарплата `< 1000` — увеличивает её на 2000 (прямо через курсор, используя `WHERE CURRENT OF`).

```sql
CREATE OR REPLACE PROCEDURE public.calculate_total_salary_2()
LANGUAGE 'plpgsql'
AS $BODY$
DECLARE
  total_salary NUMERIC := 0.00;
  the_current_cursor CURSOR FOR SELECT * FROM lecturers;
  current_record RECORD;
BEGIN
  OPEN the_current_cursor;
  LOOP
    FETCH the_current_cursor INTO current_record;
    EXIT WHEN NOT found;
    
    IF current_record.salary >= 1000 THEN
      total_salary := total_salary + current_record.salary;
    ELSE
      UPDATE lecturers 
        SET salary = current_record.salary + 2000.00
        WHERE CURRENT OF the_current_cursor;
    END IF;
  END LOOP;
  RAISE NOTICE 'Сумма равна %', total_salary;
END;
$BODY$;
```

*   **Наблюдения из лога:**
    *   После первого вызова зарплаты всех преподавателей стали равны 2800 (так как у всех была зарплата `>= 1000` после первого вызова? Нет — в логе видно, что были зарплаты 800 и 900, которые обновились, но затем данные снова менялись вручную).
    *   Процедура не только вычисляет сумму, но и **модифицирует данные** (изменяет зарплаты сотрудников с низким окладом).

## 9. Ключевые выводы и правила

1.  **Процедура vs Функция:**
    *   Процедура (`PROCEDURE`) вызывается через `CALL` и не возвращает значения.
    *   Функция (`FUNCTION`) вызывается в выражении (`SELECT`) и обязательно возвращает значение (`RETURNS`).

2.  **RAISE NOTICE:** Используется для вывода сообщений в консоль (отладка, логирование).

3.  **Курсоры:** Необходимы, когда нужно обрабатывать строки результата по одной. Явный курсор дает больше контроля (позиционирование, обновление текущей строки).

4.  **`WHERE CURRENT OF`:** Позволяет обновить или удалить именно ту строку, на которую указывает курсор. Это безопасно и удобно при итеративной обработке.

5.  **Обработка ошибок:** В примере с функцией `calculate_total_salary()` использовалась конструкция `RAISE EXCEPTION` для генерации ошибки, что видно по сообщению: *«ОШИБКА: Это сгенерированная ошибка: 7135.56»*.

6.  **Порядок создания:** Если процедура вызывает другую процедуру или функцию, вызываемый объект должен существовать на момент создания вызывающего.
