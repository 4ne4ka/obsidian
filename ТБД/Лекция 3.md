# Конспект: SQL — Скалярные и агрегатные функции

## 1. Классификация функций в SQL

Функции в SQL — это мощный инструмент для вычислений, преобразования данных и генерации результатов.

| Тип функции | Описание | Возвращаемое значение |
| :--- | :--- | :--- |
| **Скалярные функции (Scalar)** | Выполняют операции над данными одной строки. | Одно значение на каждую обработанную строку. |
| **Мультистрочные / Агрегатные функции (Aggregate)** | Выполняют операции над набором строк (группой). | Одно значение на всю группу строк. |
| **Аналитические функции (Analytical)** | Специальный тип оконных функций, ориентированных на анализ столбцов. | Значение для каждой строки с учетом окна. |

## 2. Скалярные функции (Scalar Functions)

Скалярные функции обрабатывают каждую строку индивидуально и возвращают один результат для каждой строки. Они могут быть вложенными.

### 2.1. Символьные функции (Character Functions)
*(Примеры в презентации представлены визуально, без детального списка. Обычно включают:)*
*   `UPPER()`, `LOWER()`, `INITCAP()` — преобразование регистра.
*   `CONCAT()`, `||` — конкатенация.
*   `SUBSTR()` — извлечение подстроки.
*   `LENGTH()` — длина строки.
*   `INSTR()` — позиция подстроки.
*   `LPAD()`, `RPAD()` — дополнение строки символами.
*   `TRIM()` — удаление символов по краям.

### 2.2. Числовые функции (Numeric Functions)
Основные числовые функции:

| Функция | Описание | Пример (Oracle) |
| :--- | :--- | :--- |
| **`ROUND(n, d)`** | Округление числа `n` до `d` десятичных знаков. | `ROUND(45.923, 2)` → `45.92`<br>`ROUND(45.923, 0)` → `46`<br>`ROUND(45.923, -1)` → `50` |
| **`TRUNC(n, d)`** | Усечение числа `n` до `d` десятичных знаков (без округления). | `TRUNC(45.923, 2)` → `45.92`<br>`TRUNC(45.923, 0)` → `45`<br>`TRUNC(45.923, -1)` → `40` |
| **`MOD(n, m)`** | Остаток от деления `n` на `m`. | — |
| **`ABS(n)`** | Абсолютное значение. | — |

### 2.3. Функции для работы с датами и временем

#### Операции с датами (PostgreSQL)
В PostgreSQL даты поддерживают арифметические операции:
*   `date + integer` → дата + количество дней.
*   `date + interval` → дата + временной интервал.
*   `date - date` → количество дней (integer).
*   `timestamp - timestamp` → интервал (interval).

**Пример (Oracle):** Вычисление количества недель работы сотрудника.
```sql
SELECT ename, (SYSDATE - hiredate) / 7 AS WEEKS
FROM emp
WHERE deptno = 10;
```

#### Основные функции для дат (Oracle/PostgreSQL)

| Функция | Описание (Oracle) | Аналог / Описание (PostgreSQL) |
| :--- | :--- | :--- |
| `SYSDATE` | Текущая дата и время. | `CURRENT_DATE`, `CURRENT_TIMESTAMP`, `NOW()` |
| `MONTHS_BETWEEN(d1, d2)` | Количество месяцев между датами. | `AGE(d1, d2)` — возвращает интервал. |
| `ADD_MONTHS(d, n)` | Прибавляет `n` месяцев к дате. | `d + INTERVAL 'n months'` |
| `LAST_DAY(d)` | Последний день месяца для даты `d`. | `DATE_TRUNC('month', d) + INTERVAL '1 month - 1 day'` |
| `NEXT_DAY(d, 'day')` | Следующая указанная дата (например, следующий понедельник). | — |
| `EXTRACT(field FROM source)` | Извлечение части даты (год, месяц и т.д.). | `EXTRACT(field FROM source)` |
| `DATE_TRUNC(precision, source)` | — | Усечение даты до указанной точности (например, `'hour'`). |

**Пример (PostgreSQL):**
```sql
-- Извлечение часа из временной метки
SELECT EXTRACT(hour FROM TIMESTAMP '2001-02-16 20:38:40'); -- Результат: 20

-- Усечение до начала часа
SELECT DATE_TRUNC('hour', TIMESTAMP '2001-02-16 20:38:40'); -- Результат: 2001-02-16 20:00:00
```

### 2.4. Функции преобразования типов данных (Conversion)

#### В Oracle
*   **`TO_CHAR(date, 'format')`** — преобразует дату в строку по формату.
    *   Формат регистрозависим!
    *   Элементы формата: `YYYY` (год цифрами), `YEAR` (год строкой), `MM` (месяц цифрами), `MONTH` (название месяца), `DD` (день), `DY` (день недели аббр.), `DAY` (день недели).
    ```sql
    SELECT ename, TO_CHAR(hiredate, 'fmDD Month YYYY') AS hiredate FROM emp;
    -- Результат: KING, 17 November 1981
    ```
*   **`TO_CHAR(number, 'format')`** — преобразует число в строку по формату.
    *   Элементы формата: `9` (цифра), `0` (цифра с ведущим нулем), `$` (знак доллара), `L` (локальная валюта), `.` (десятичная точка), `,` (разделитель тысяч).
    ```sql
    SELECT TO_CHAR(sal, '$99,999') AS salary FROM emp WHERE ename = 'SCOTT';
    -- Результат: $3,000
    ```
*   **`TO_DATE(string, 'format')`** — преобразует строку в дату.
*   **`TO_NUMBER(string, 'format')`** — преобразует строку в число.

#### В PostgreSQL
*   **Операторы приведения:**
    *   `CAST(expression AS type)`
    *   `expression::type`
    ```sql
    SELECT 123::real;  -- Преобразует целое число 123 в тип real
    ```
*   **Функции форматирования:**
    *   `to_char(timestamp, text)`
    *   `to_date(text, text)`
    *   `to_number(text, text)`
    *   `to_timestamp(text, text)`

## 3. Агрегатные (мультистрочные) функции

Агрегатные функции работают с набором строк и возвращают одно единственное значение.

### 3.1. Основные агрегатные функции

| Функция | Описание | Примечание |
| :--- | :--- | :--- |
| **`SUM(expr)`** | Сумма значений `expr` по группе. | Только для числовых данных. |
| **`AVG(expr)`** | Среднее арифметическое значений `expr`. | Только для числовых данных. |
| **`MIN(expr)`** | Минимальное значение `expr` по группе. | Работает с любыми типами данных (даты, строки). |
| **`MAX(expr)`** | Максимальное значение `expr` по группе. | Работает с любыми типами данных. |
| **`COUNT(*)`** | Количество всех строк в группе (включая NULL). | Возвращает общее количество строк. |
| **`COUNT(expr)`** | Количество не-NULL значений `expr` в группе. | Игнорирует строки, где `expr = NULL`. |

**Пример:**
```sql
SELECT AVG(sal), MAX(sal), MIN(sal), SUM(sal)
FROM emp
WHERE job LIKE 'SALES%';
-- Результат: AVG=1400, MAX=1600, MIN=1250, SUM=5600
```

### 3.2. Обработка NULL в агрегатных функциях
*   **Агрегатные функции (кроме `COUNT(*)`) игнорируют значения NULL.**
*   Если нужно заменить NULL перед агрегацией, используется функция `NVL` (Oracle) или `COALESCE` (стандарт SQL).

### 3.3. Группировка данных — `GROUP BY`

Предложение `GROUP BY` делит строки на группы, к которым затем применяются агрегатные функции.

**Синтаксис:**
```sql
SELECT column, group_function(column)
FROM table
[WHERE condition]
[GROUP BY group_by_expression]
[ORDER BY column];
```

**Важное правило:**
> Любой столбец или выражение в списке `SELECT`, **не являющееся аргументом агрегатной функции**, должно быть включено в предложение `GROUP BY`.

*   **Группировка по одному полю:**
    ```sql
    SELECT deptno, AVG(sal) AS avg_salary
    FROM emp
    GROUP BY deptno;
    ```
*   **Группировка по нескольким полям:**
    ```sql
    SELECT deptno, job, SUM(sal)
    FROM emp
    GROUP BY deptno, job;
    ```

### 3.4. Фильтрация групп — `HAVING`

Предложение `HAVING` используется для фильтрации групп после применения агрегатных функций (в отличие от `WHERE`, который фильтрует строки до группировки).

**Порядок следования предложений (обязателен!):**
1.  `SELECT`
2.  `FROM`
3.  `WHERE` — фильтрация строк *до* группировки.
4.  `GROUP BY` — формирование групп.
5.  `HAVING` — фильтрация групп *после* группировки.
6.  `ORDER BY` — сортировка результата.

**Пример:**
```sql
-- Вывести должности (кроме SALES) с суммарной зарплатой > 5000
SELECT job, SUM(sal) AS PAYROLL
FROM emp
WHERE job NOT LIKE 'SALES'      -- Фильтрация строк до группировки
GROUP BY job
HAVING SUM(sal) > 5000          -- Фильтрация групп после агрегации
ORDER BY SUM(sal);
```

### 3.5. Вложенные агрегатные функции

Агрегатные функции могут быть вложенными. В этом случае внутренняя функция вычисляется для каждой группы, а внешняя — для результата внутренней.

*   **Пример:** Найти максимальное значение средних зарплат по отделам.
    ```sql
    SELECT MAX(AVG(sal))
    FROM emp
    GROUP BY deptno;
    ```

### 3.6. Порядок выполнения оператора SELECT с агрегатными функциями

1.  **FROM** — определение источника данных.
2.  **WHERE** — фильтрация строк.
3.  **GROUP BY** — разделение на группы.
4.  **HAVING** — фильтрация групп.
5.  **SELECT** — вычисление выражений и агрегатных функций.
6.  **ORDER BY** — сортировка результата.

## 4. Агрегатные функции PostgreSQL (расширенный список)

PostgreSQL поддерживает стандартные функции и добавляет специализированные:

| Функция | Описание |
| :--- | :--- |
| `AVG()`, `SUM()`, `MIN()`, `MAX()` | Стандартные агрегатные функции. |
| `COUNT(*)`, `COUNT(expr)` | Подсчет строк. |
| `BIT_AND(expr)` | Побитовое `AND` всех не-NULL входных значений. |
| `BIT_OR(expr)` | Побитовое `OR` всех не-NULL входных значений. |
| `BOOL_AND(expr)` | Возвращает `true`, если все входные значения `true`. |
| `BOOL_OR(expr)` | Возвращает `true`, если хотя бы одно входное значение `true`. |
| `EVERY(expr)` | Эквивалент `BOOL_AND`. |
| `STRING_AGG(expr, delimiter)` | Объединяет не-NULL значения в строку, разделяя их указанным разделителем. |
