## 2. Философия: логика на клиенте или на сервере?

- Единого ответа нет.
- Всю бизнес-логику и обработку данных **можно сделать на сервере**, так как процедурные языки (PL/SQL, PL/pgSQL, T-SQL) достаточно мощные.
- Два подхода к решению задач обработки данных (особенно аналитической):
  1. **Декларативный SQL (SELECT)** — мощный, оптимизируемый.
  2. **Процедурный код** — привычный, с циклами, условиями, функциями.

### Главный принцип (из опыта):
> Если задачу можно решить **оператором SELECT** — надо решать SELECT.  
> Процедурные расширения использовать только там, где SELECT слишком сложен или невозможен.

**Почему SELECT лучше:**
- Он — объект оптимизации **оптимизатора SQL** (математические основы, дискретная математика).
- Оптимизатор выжимает максимум производительности.
- Можно помочь оптимизатору (позже будет тема).

## 4. PL/SQL и аналоги — общая характеристика

## 5. Типы программных единиц (хранимых объектов)

### 1. Процедура
- Не возвращает значение (через RETURN).
- Может изменять данные.
- Вызывается явно.

### 2. Функция
- **Обязательно возвращает значение**.
- Вызывается **внутри выражения** (например, в SELECT).

### 3. Триггер
- В PostgreSQL — это функция, вызываемая **автоматически** при событии с данными (INSERT, UPDATE, DELETE).
- Нельзя вызвать вручную.
- События только с записями (не со схемой).

## 6. Объявление переменных

**Важное отличие от C/Java:**  
- Переменные объявляются **по одной**.
- Синтаксис:
  ```sql
  имя тип [CONSTANT] [NOT NULL] [DEFAULT выражение | := выражение]
  ```

**Пример:**
```sql
DECLARE
  pi CONSTANT NUMERIC := 3.1415926;
  total NUMERIC DEFAULT 0;
```

- **CONSTANT** — нельзя изменять.
- **DEFAULT** или `:=` — инициализация.
- Если NOT NULL и нет инициализации — будет исключение.

## 8. Циклы (стиль Ada)

### 1. Бесконечный цикл LOOP
```sql
LOOP
  -- действия
  EXIT; -- выход
END LOOP;
```

**С условием EXIT WHEN:**
```sql
LOOP
  EXIT WHEN x > 100;
  x := x + 1;
END LOOP;
```

### 2. Цикл WHILE
```sql
WHILE условие LOOP
  ...
END LOOP;
```

### 3. Цикл FOR (со счётчиком)
```sql
FOR i IN 1..100 LOOP
  total := total + i;
END LOOP;
```

- `REVERSE` — обратный порядок.
- Шаг всегда 1 (или -1 с REVERSE), изменить шаг нельзя.

## 9. Практический пример: процедура на PL/pgSQL (PostgreSQL)

**Задача:** сумма арифметической прогрессии от 1 до 100.

```sql
CREATE OR REPLACE PROCEDURE test_progression()
LANGUAGE plpgsql
AS $$
DECLARE
  s1 NUMERIC := 0;
BEGIN
  FOR i IN 1..100 LOOP
    s1 := s1 + i;
  END LOOP;
  RAISE NOTICE 'Сумма прогрессии = %', s1;
END;
$$;
```

**Вызов:**
```sql
CALL test_progression();
```
Результат: 5050.

## 10. Работа с данными — курсоры

### Что такое курсор
- Указатель на **текущую строку** результата SELECT.
- Позволяет построчно обрабатывать набор записей.

### Неявный курсор (FOR ... LOOP)
Самый простой способ.

**Пример:** подсчёт суммы зарплат из таблицы lecturers.

```sql
CREATE OR REPLACE PROCEDURE calculate_total_salary()
LANGUAGE plpgsql
AS $$
DECLARE
  total_salary NUMERIC := 0;
  current_record RECORD;
BEGIN
  FOR current_record IN SELECT * FROM lecturers LOOP
    total_salary := total_salary + current_record.salary;
  END LOOP;
  RAISE NOTICE 'Полученная сумма = %', total_salary;
END;
$$;
```

- `RECORD` — специальный тип, который принимает структуру строки SELECT.
- Порядок обхода — как в SELECT. Для явного порядка добавьте `ORDER BY`.

---

## 11. Явный курсор

**Шаги:**
1. Объявить курсор (SELECT не выполняется).
2. Открыть (`OPEN`).
3. Выбирать строки (`FETCH`).
4. Закрыть (`CLOSE` — необязательно, закроется сам при завершении процедуры).

### Пример: сумма зарплат > 1000 (явный курсор)

```sql
CREATE OR REPLACE PROCEDURE calculate_total_salary2()
LANGUAGE plpgsql
AS $$
DECLARE
  total_salary NUMERIC := 0;
  current_record RECORD;
  cur CURSOR FOR SELECT * FROM lecturers WHERE salary > 1000;
BEGIN
  OPEN cur;
  LOOP
    FETCH cur INTO current_record;
    EXIT WHEN NOT FOUND;
    total_salary := total_salary + current_record.salary;
  END LOOP;
  CLOSE cur;
  RAISE NOTICE 'Сумма (явный курсор) = %', total_salary;
END;
$$;
```

- `FOUND` — системная переменная. `NOT FOUND` — конец набора.
- Переменную `FOUND` нельзя объявить — она предопределена.

---

## 12. Навигация в курсоре (опционально)

- `FETCH` может использовать `RELATIVE n` (на n строк вперёд/назад) и `ABSOLUTE n` (на строку с номером n).
- `MOVE` — только перемещение без получения данных.

Пример:
```sql
FETCH RELATIVE -2 FROM cur INTO rec;  -- на 2 строки назад
FETCH ABSOLUTE 5 FROM cur INTO rec;   -- на 5-ю строку
```

---

## 13. Обновление данных через курсор

Можно **изменять текущую строку** (если курсор обновляемый).

**Пример:** если зарплата меньше 1000 — увеличить на 2000.

```sql
CREATE OR REPLACE PROCEDURE update_salary()
LANGUAGE plpgsql
AS $$
DECLARE
  current_record RECORD;
  cur CURSOR FOR SELECT * FROM lecturers FOR UPDATE;
BEGIN
  OPEN cur;
  LOOP
    FETCH cur INTO current_record;
    EXIT WHEN NOT FOUND;
    IF current_record.salary < 1000 THEN
      UPDATE lecturers
      SET salary = salary + 2000
      WHERE CURRENT OF cur;   -- обновить текущую строку курсора
    END IF;
  END LOOP;
  CLOSE cur;
END;
$$;
```

**Важно:**
- `FOR UPDATE` при объявлении курсора.
- `WHERE CURRENT OF курсор` — изменяет именно ту строку, на которой стоит курсор.

## 14. Результат выполнения примера (иллюстрация)

Было:
- Лектор А: 800  
- Лектор Б: 1200  

После обновления:
- Лектор А: 2800  
- Лектор Б: 1200 (не изменился, т.к. условие <1000 не выполнено)

## Итоговые тезисы

1. **SELECT** предпочтительнее процедурного кода, где возможно.
2. **Процедуры и функции** пишутся на PL/pgSQL (PostgreSQL) или аналогах.
3. **Переменные** объявляются по одной, инициализация через `DEFAULT` или `:=`.
4. **Циклы**: LOOP, WHILE, FOR (счётчик — только целые, шаг 1).
5. **Неявный курсор** (`FOR rec IN SELECT ... LOOP`) — самый удобный для построчной обработки.
6. **Явный курсор** даёт больше контроля: OPEN, FETCH, CLOSE, проверка `FOUND`.
7. Через курсор можно **обновлять** текущую строку (`UPDATE ... WHERE CURRENT OF`).
