# Конспект: Реляционная модель и основы SQL

## 1. Реляционная модель данных

### 1.1. Основные принципы
*   **Хранение данных:** Данные хранятся в виде прямоугольных плоских таблиц (Flat Tables).
*   **Связи:** Связи между таблицами реализуются исключительно через значения данных (например, через внешние ключи), а не через указатели.

### 1.2. Определение реляционной таблицы
Таблица (Relation) должна соответствовать следующим свойствам:
1.  **Строки (Rows):** Каждая строка соответствует одному информационному объекту (сущности) и описывает его.
2.  **Столбцы (Columns):** Каждый столбец соответствует **атомарному** атрибуту объекта и имеет уникальное имя.
3.  **Уникальность столбцов:** Столбцы не дублируются и описывают разные свойства объектов.
4.  **Уникальность строк:** В таблице нет дублирующихся строк.
5.  **Неупорядоченность:** Перестановка двух строк и/или двух столбцов не меняет смысла данных.

### 1.3. Характеристики значений
*   **Домен (Domain):** Множество допустимых значений для атрибута.
*   **Атомарность (Atomic values):** Значения атрибутов должны быть неделимыми (не массив, не составной объект). *Примечание: это требование нормальных форм, хотя современные СУБД поддерживают и сложные типы (JSON, массивы).*
*   **NULL:** Специальное значение, входящее в любой домен. Означает "неизвестно", "не определено" или "не применимо". NULL — это не ноль и не пробел.

### 1.4. Схема и экземпляр отношения
*   **Схема отношения (Relation Schema):** Имя отношения и набор атрибутов. *Пример: `instructor = (ID, name, dept_name, salary)`.*
*   **Экземпляр отношения (Relation Instance):** Набор кортежей (строк), удовлетворяющих схеме на данный момент времени.

### 1.5. Ключи
*   **Суперключ (Superkey):** Множество атрибутов, достаточное для уникальной идентификации кортежа. *Пример: {ID}, {ID, name}.*
*   **Кандидат в ключи (Candidate Key):** Минимальный суперключ (нельзя удалить ни один атрибут без потери уникальности). *Пример: {ID}.*
*   **Первичный ключ (Primary Key):** Кандидат в ключи, выбранный в качестве основного идентификатора.

### 1.6. Внешний ключ (Foreign Key)
*   **Определение:** Значение в одной таблице (ссылающаяся) должно соответствовать значению в другой таблице (ссылаемая).
*   **Правила ссылочной целостности:** Определяют поведение при удалении или изменении записи в ссылаемой таблице.
    *   `NO ACTION` (или `RESTRICT`): Запрещает действие, если есть зависимые записи.
    *   `CASCADE`: Каскадно удаляет или изменяет зависимые записи.
    *   `SET NULL`: Устанавливает NULL в зависимых записях.
    *   `SET DEFAULT`: Устанавливает значение по умолчанию.

## 2. Основы SQL

### 2.1. История SQL
*   **1970:** Доктор Эдгар Кодд публикует реляционную модель.
*   **1978:** IBM разрабатывает System/R и язык SEQUEL (Structured English Query Language).
*   **1979:** Компания Relational Software (позже Oracle) выпускает первую коммерческую RDBMS с SQL.
*   **1986-...:** Стандартизация ANSI/ISO: SQL-87, SQL-92 (SQL2), SQL:1999 (SQL3), SQL:2003, SQL:2006, SQL:2008, SQL:2011.

### 2.2. Классификация команд SQL
*   **DML (Data Manipulation Language):** `INSERT`, `UPDATE`, `DELETE`.
*   **DDL (Data Definition Language):** `CREATE`, `ALTER`, `DROP`, `RENAME`, `TRUNCATE`.
*   **TCL (Transaction Control Language):** `COMMIT`, `ROLLBACK`, `SAVEPOINT`.
*   **DCL (Data Control Language):** `GRANT`, `REVOKE`.

## 3. SELECT — Оператор запроса

### 3.1. Синтаксис и порядок выполнения
**Базовый синтаксис:**
```sql
SELECT [DISTINCT] {*, column [alias], ...}
FROM table
[WHERE condition(s)]
[ORDER BY column [ASC|DESC]];
```

**Порядок логического выполнения (важно):**
1.  **FROM:** Определение таблицы-источника.
2.  **WHERE:** Фильтрация строк (условия).
3.  **SELECT:** Вычисление выражений, применение DISTINCT, псевдонимы.
4.  **ORDER BY:** Сортировка результата (выполняется последней).

### 3.2. SELECT и FROM
*   `SELECT * FROM dept;` — выбор всех столбцов.
*   `SELECT deptno, loc FROM dept;` — выбор конкретных столбцов.

### 3.3. Псевдонимы (Aliases)
*   Переименовывают заголовок столбца в результате.
*   Ключевое слово `AS` опционально.
*   Двойные кавычки (`" "`) обязательны, если псевдоним содержит пробелы или регистрозависим.

```sql
SELECT ename AS name, sal * 12 "Yearly Salary" FROM emp;
```

### 3.4. Конкатенация (||)
*   Объединяет строковые значения.
*   Строковые литералы заключаются в одинарные кавычки (`' '`).

```sql
SELECT ename || ' works as a ' || job AS "Employee Details" FROM emp;
```

### 3.5. Удаление дубликатов (DISTINCT)
*   По умолчанию SELECT возвращает все строки (включая дубликаты).
*   `SELECT DISTINCT job FROM emp;` — убирает повторяющиеся значения.

### 3.6. WHERE — Фильтрация строк
**Операторы сравнения:**
`=`, `>`, `>=`, `<`, `<=`, `<>` (или `!=` — не равно).

**Специальные операторы:**
*   **`BETWEEN ... AND ...`** — диапазон (включая границы).
    ```sql
    SELECT * FROM emp WHERE sal BETWEEN 1000 AND 2000;
    ```
*   **`IN (...)`** — проверка на вхождение в список.
    ```sql
    SELECT * FROM emp WHERE deptno IN (10, 20);
    ```
*   **`LIKE`** — поиск по шаблону.
    *   `%` — любое количество символов (включая ноль).
    *   `_` — ровно один символ.
    ```sql
    SELECT ename FROM emp WHERE ename LIKE 'S%';   -- Имена на S
    SELECT ename FROM emp WHERE ename LIKE '_A%'; -- Второй символ A
    ```
*   **`IS NULL`** — проверка на NULL. *Важно: `= NULL` не работает.*
    ```sql
    SELECT * FROM emp WHERE comm IS NULL;
    ```

### 3.7. Логические операторы
*   **`AND`:** Истина, если оба условия истинны.
*   **`OR`:** Истина, если хотя бы одно условие истинно.
*   **`NOT`:** Инверсия (отрицание). Применяется к операторам (`NOT IN`, `NOT LIKE`, `NOT BETWEEN`).

**Приоритет (сверху вниз):**
1.  Операторы сравнения
2.  **NOT**
3.  **AND**
4.  **OR**

```sql
-- Используйте скобки для изменения приоритета
WHERE (job = 'SALESMAN' OR job = 'PRESIDENT') AND sal > 1500;
```

### 3.8. ORDER BY — Сортировка
*   **`ASC`:** По возрастанию (по умолчанию).
*   **`DESC`:** По убыванию.
*   Выполняется последней.
*   Можно сортировать по столбцам, не включенным в `SELECT`.

```sql
SELECT ename, hiredate FROM emp ORDER BY hiredate DESC;
```

### 3.9. Ограничение количества строк (нестандартное)
В презентации упоминается синтаксис `TOP` (характерно для SQL Server), но в PostgreSQL используется `LIMIT` (хотя в слайдах показан `TOP`).

```sql
-- Пример из слайда (T-SQL стиль)
SELECT TOP 2 ename, sal FROM emp WHERE job = 'MANAGER' ORDER BY sal DESC;

-- Аналог в PostgreSQL:
SELECT ename, sal FROM emp WHERE job = 'MANAGER' ORDER BY sal DESC LIMIT 2;
```

## 4. NULL в выражениях
*   Любое арифметическое выражение, содержащее `NULL`, возвращает `NULL`.
    ```sql
    -- Если COMM = NULL, то 12 * sal + comm = NULL
    SELECT ename, 12 * sal + comm FROM emp;
    ```

## 5. Типы данных (PostgreSQL)

### 5.1. Числовые (Numeric)
*   `SMALLINT`, `INTEGER`, `BIGINT`
*   `DECIMAL(p,s)`, `NUMERIC(p,s)` — точные числа с заданной точностью.
*   `REAL`, `DOUBLE PRECISION` — числа с плавающей точкой.
*   `SMALLSERIAL`, `SERIAL`, `BIGSERIAL` — автоинкрементные псевдотипы (для первичных ключей).
*   `MONEY` — для валют.

### 5.2. Символьные (Character)
*   `CHAR(n)` / `CHARACTER(n)` — фиксированная длина, дополняется пробелами.
*   `VARCHAR(n)` / `CHARACTER VARYING(n)` — переменная длина с ограничением.
*   `TEXT` — переменная длина без ограничения.

### 5.3. Бинарные
*   `BYTEA` — для хранения бинарных строк (изображения, файлы).

### 5.4. Дата и время
*   `DATE` — только дата.
*   `TIME` — только время.
*   `TIMESTAMP` — дата и время.
*   `INTERVAL` — интервал времени.

### 5.5. Логический
*   `BOOLEAN` — значения `TRUE`, `FALSE`, `NULL`.

### 5.6. Специализированные типы
*   **`ENUM`:** Перечисляемый тип (создается через `CREATE TYPE`).
    ```sql
    CREATE TYPE mood AS ENUM ('sad', 'ok', 'happy');
    CREATE TABLE person ( name text, current_mood mood );
    ```
*   **Геометрические:** `POINT`, `LINE`, `CIRCLE`, `POLYGON` и др.
*   **Сетевые:** `INET` (IPv4/IPv6), `MACADDR`.
*   **Битовые:** `BIT(n)`, `BIT VARYING(n)`.
*   **JSON/XML:** `JSON`, `JSONB`, `XML`.
*   **Массивы:** Поддержка многомерных массивов любых типов.
*   **UUID:** Универсальный уникальный идентификатор.

## 6. DDL — CREATE TABLE (Пример)

```sql
CREATE TABLE weather (
    city    CHARACTER VARYING(80),
    temp_lo INTEGER,   -- низкая температура
    temp_hi INTEGER,   -- высокая температура
    prcp    REAL,      -- осадки
    date    DATE
);
```
