### Документация в Unix
Форматы документации:
- `.txt`, `.html`, `.pdf`
- Специфичные для Unix: **man** и **info**

### Навигация в man-страницах
- Стрелки вверх/вниз, PageUp/PageDown, Home/End (работают не везде).
- Выход — клавиша **q**.

### Структура man-страницы
1. **NAME** — краткое описание (одна строка). По этому разделу потом выполняется поиск (`apropos`, `whatis`).
2. **SYNOPSIS** — синтаксис:
   - Для программ: формат командной строки.
   - Для функций: прототип, подключаемые заголовочные файлы.
   - Обозначения:
     - Обязательные параметры — без скобок.
     - `[ ]` — необязательные параметры.
     - `{ }` — обязательные, но с выбором вариантов (через `|`).
     - `разделитель...` — параметр можно повторить несколько раз (разделитель чаще пробел, реже запятая).
3. **DISCLAIMER / DESCRIPTION** — подробное описание (несколько страниц).
4. **OPTIONS** — все аргументы командной строки (у многих программ их 20+). Примеры программ с большим числом параметров: **Git**, **компилятор**.
5. **CATPAGES / SEE ALSO** — ссылки на другие разделы документации.
6. **ENVIRONMENT** — переменные окружения, влияющие на работу программы.

### Регистр символов в Unix
- Большие и маленькие буквы — **разные** символы.
- Пример: `a.txt`, `A.txt`, `a.TXT` — три разных файла.
- В Windows регистр не учитывается.
- В параметрах команд часто маленькая буква означает одно, большая — противоположное (например, сортировка по возрастанию / убыванию).

### Переменные окружения
- Команда `env` — выводит все переменные окружения.
- Формат: `ИМЯ=значение`.
- Переменные наследуются дочерними процессами.
- В программе на C доступ через `getenv()`/`setenv()` (из любой функции). В отличие от аргументов командной строки, которые нужно явно передавать через параметры функций.
- Если параметр можно задать и аргументом, и переменной окружения — приоритет у аргумента командной строки.

### Работа с несколькими консолями
- **Alt+F1 … Alt+F6** — переключение между локальными консолями.
- В каждой консоли можно открыть отдельную сессию (того же или другого пользователя).
- Прокрутка вывода, который не поместился на экране: **Shift+PgUp** / **Shift+PgDn** (не на цифровой клавиатуре).

### Завершение сессии
Три способа:
- `exit`
- `logout`
- **Ctrl+D** — означает «конец ввода», командный интерпретатор закрывается.

### Поиск по документации
- **`apropos термин`** — ищет подстроку в разделе NAME всех man-страниц. Если термин короткий (например, `man`), результат может быть огромным.
- **`whatis термин`** — ищет точное совпадение слова (а не подстроку) в разделе NAME.
- Внутри `man` можно искать по всем страницам с помощью ключа (например, `man -K строка`).

### Номера разделов man
Один термин может существовать в разных разделах:
- **1** — программы (пользовательские команды)
- **2** — системные вызовы
- **3** — функции языка C
- **4** — специальные файлы (устройства)
- **5** — конфигурационные файлы
- **6** — игры
- **7** — общие термины (например, `signal`)
- **8** — команды администратора
- **9** — ядро

Примеры:
- `man 2 signal` — системный вызов
- `man 7 signal` — список сигналов Linux с их действиями по умолчанию

### Информация о сигналах
- Сигналы — механизм межпроцессного взаимодействия.
- `Ctrl+C` — обычно SIGINT.
- `Ctrl+Z` — SIGTSTP (приостановка процесса). Лектор предупреждает: в системах общего пользования приостанавливать процессы нехорошо.

### Info-документация
- Команда `info термин` — гипертекстовая документация.
- Если набрать просто `info` — откроется документ по умолчанию.
- Переход по ссылке — **Enter**.
- Есть свои команды навигации (много сочетаний клавиш).
- Лектор предпочитает `man`, но иногда информация в `man` и `info` различается.

### Завершение лекции
- **`shutdown`** — перевод системы на другой уровень выполнения.
- Обязательный параметр — **время** (когда выполнить переход). Без указания уровня переходит на уровень **1**.
- Уровни выполнения (runlevels):
  - **0** — выключение
  - **1** — однопользовательский режим
  - **2** — многопользовательский текстовый (без некоторых сетевых служб)
  - **3** — полный многопользовательский текстовый
  - **4** — не используется по умолчанию (пользовательский)
  - **5** — графический режим
  - **6** — перезагрузка
  - **S** (или `s`) — тоже однопользовательский

## Подробное описание каждой команды

### 1. `man`
**Назначение:** просмотр справочных страниц (manual pages) в Unix.
**Синтаксис:** `man [номер_раздела] термин [термин2 ...]`
**Примеры:**
- `man man` — справка о самой команде `man`.
- `man 2 signal` — справка о системном вызове `signal` из раздела 2.
- `man 7 signal` — список сигналов из раздела 7.
**Особенности:**
- Если указано несколько терминов, страницы открываются последовательно (после выхода из первой — вторая и т.д.).
- Внутри используется пейджер (обычно `less`).
- **Навигация:** стрелки, PgUp/PgDn, поиск (`/`), выход (`q`).
- Поддерживает русский язык для многих программ.

### 2. `apropos`
**Назначение:** поиск по кратким описаниям (раздел NAME) всех man-страниц. Ищет **подстроку**.
**Синтаксис:** `apropos термин`
**Пример:** `apropos fstab` — найдёт все страницы, в описании которых встречается «fstab».
**Опасность:** `apropos man` выдаст очень много строк (man есть в словах «command», «management» и т.д.).

### 3. `whatis`
**Назначение:** поиск **точного совпадения** слова (а не подстроки) в разделе NAME.
**Синтаксис:** `whatis термин`
**Пример:** `whatis man` — выдаст всего несколько строк, где «man» — отдельное слово.

### 4. `env`
**Назначение:** показать все текущие переменные окружения.
**Синтаксис:** `env`
**Пример вывода:** `PATH=/usr/bin`, `HOME=/root`, `LANG=ru_RU.UTF-8` и т.д.
**Применение:** также может использоваться для запуска программы с изменённым окружением (например, `env DISPLAY=:0 xclock`).

### 5. `exit`, `logout`, **Ctrl+D**
**Назначение:** завершить сессию командной оболочки.
- `exit` — завершает текущий экземпляр оболочки.
- `logout` — завершает сессию (работает только для login shell).
- `Ctrl+D` — отправляет оболочке признак конца файла (EOF), оболочка закрывается.

### 6. `info`
**Назначение:** просмотр гипертекстовой документации (обычно более подробной, чем man).
**Синтаксис:** `info [термин]`
**Пример:** `info` — открывает главное меню; `info man` — документация по man в формате info.
**Навигация:**  
- **Enter** — переход по ссылке (ссылка отмечена `*`).  
- **u** — на уровень вверх.  
- **p** — предыдущая страница.  
- **n** — следующая.  
- **q** — выход.

### 7. `shutdown`
**Назначение:** остановка, перезагрузка или перевод системы в другой уровень выполнения (runlevel).
**Синтаксис:** `shutdown [опции] время [сообщение]`
**Обязательный параметр:** время (например, `now`, `+10`, `23:00`).
**Примеры:**
- `shutdown now` — выключение (если уровень не указан, по умолчанию переход на уровень 1, но в современных системах — выключение).
- `shutdown -r now` — перезагрузка.
- `shutdown -h now` — выключение питания (halt).
- `shutdown -c` — отмена запланированного выключения.
**Уровни выполнения (runlevels):**  
0 — выкл.  
1 — однопользовательский (администрирование).  
2-4 — многопользовательские текстовые (конкретное значение зависит от дистрибутива).  
5 — многопользовательский с графикой (X Window).  
6 — перезагрузка.  
S — однопользовательский (без монтирования всех файловых систем).

## Дополнительные сведения (из лекции)

### Unix Way
- Каждая программа делает **одно дело**, но делает его хорошо.
- Программы комбинируются в **конвейеры** (pipe, `|`), где вывод одной подаётся на вход другой.
- Большое количество параметров объясняется:
  - Поддержкой редких случаев (например, подготовка man-страницы к печати).
  - Необходимостью точно управлять форматом вывода для последующей обработки другими программами.

### Аргументы командной строки vs переменные окружения
| Аргументы командной строки | Переменные окружения |
|----------------------------|----------------------|
| Передаются явно при запуске | Наследуются от родителя |
| Приоритет выше | Используются, если аргумент не задан |
| Требуют передачи через цепочку вызовов | Доступны в любой функции (`getenv`) |
| Пример: `gcc -O2 file.c` | Пример: `CFLAGS=-O2 gcc file.c` |
