### 1. Программная модель процессора (ISA - Instruction Set Architecture)

#### 1.1. Места хранения данных

*   **Регистры центрального процессора (ЦП)** — быстрая, но небольшая память внутри ЦП.
    *   **Целочисленные регистры (R0...R15):** 16 регистров для хранения 32-битных целых чисел.
    *   **Вещественные регистры (F0...F15):** 16 регистров для хранения 64-битных чисел двойной точности (формат `double`).
    *   **Специальные регистры:**
        *   **IP (Instruction Pointer) / PC (Program Counter):** 32-битный регистр, содержащий адрес следующей инструкции для выполнения.
        *   **FLAGS (регистр флагов):** Регистр, в котором каждый бит (или группа битов) имеет специальное значение (например, "был ли результат нулевым", "было ли переполнение"). Эти флаги выставляются после арифметических операций и операций сравнения (`CMP`) и используются для условных переходов.

    *Важное замечание:* В реальных архитектурах регистры часто имеют специализацию, но в нашей модели все `R0-R15` и `F0-F15` считаются однородными.
*   **Оперативная память (ОЗУ):**
    *   Адресуется с помощью 32-битных адресов.
    *   Процессор может читать (`LD`) и записывать (`ST`) данные порциями по 32 бита (в целочисленные регистры) или 64 бита (в вещественные регистры).
### 2. Набор инструкций (Instruction Set)
#### 2.1. Инструкции доступа к памяти
*   **`LD [address], register`** — загрузить данные из памяти по указанному адресу в регистр.
*   **`ST register, [address]`** — сохранить данные из регистра в память по указанному адресу.
*   **`MOV register1, register2`** — скопировать значение из `register1` в `register2`.
*   **`PUSH` / `POP`** — работа со стеком (будут рассмотрены позже).

*Ключевой момент:* Рассматриваемая архитектура является **Load/Store архитектурой**. Это означает, что арифметические операции (сложение, умножение и т.д.) могут выполняться **только над регистрами**.
#### 2.2. Арифметические инструкции
Формат инструкций: `Операция источник1, источник2, результат`. Все операнды — **регистры**.
*   **Целочисленные операции (для регистров R0-R15):**
    *   `ADD R1, R2, R3`  // R3 = R1 + R2
    *   `SUB R1, R2, R3`  // R3 = R1 - R2
    *   `MUL R1, R2, R3`  // R3 = R1 * R2
    *   `DIV R1, R2, R3, R4` // R3 = R1 / R2 (частное), R4 = R1 % R2 (остаток)
    *   `CMP R1, R2`       // Вычислить R1 - R2, результат не сохраняется, но **устанавливаются флаги** в регистре `FLAGS`.
    *   `AND, OR, NOT, XOR` — побитовые операции.
    *   `SHL, SHR` — побитовые сдвиги (в нашей модели — арифметические, т.е. знаковый бит сохраняется).
*   **Вещественные операции (для регистров F0-F15):**
    *   `FADD F1, F2, F3`  // F3 = F1 + F2
    *   `FSUB, FMUL, FDIV` — аналогично.
    *   `FCMP` — сравнение вещественных чисел с установкой флагов.

*Важное замечание:* Целочисленные и вещественные регистры **нельзя смешивать** в одной операции. `ADD R1, F2, R3` — недопустимо.

#### 2.3. Инструкции управления потоком выполнения
*   **`JMP [address]`** — **безусловный переход**. Перемещает выполнение на указанный адрес (меняет значение `IP`).
*   **Условные переходы** — выполняют переход, только если условие, основанное на флагах (обычно от последней операции `CMP`), истинно.
    *   `JE [address]`  — Jump if Equal (равны)
    *   `JNE [address]` — Jump if Not Equal (не равны)
    *   `JG [address]`  — Jump if Greater (больше)
    *   `JGE [address]` — Jump if Greater or Equal (больше или равно)
    *   `JL [address]`  — Jump if Less (меньше)
    *   `JLE [address]` — Jump if Less or Equal (меньше или равно)

*   **`CALL` / `RET`** — инструкции для вызова функций и возврата из них (будут рассмотрены позже).
`JMP` используем абсолютные адреса
`JХХ` используем относительные адреса (+4, -4)
### 3. Пример: Карта памяти и код
**Карта памяти:**
| Адрес | Переменная | Значение |
| 1000 | `i` | `0` |
| 1004 | `N` | `10` |
| 1008 | `Res` | `0.0` |
| 1016 | `1.0` | `1.0` |
**Эквивалентный код на C:**
```c
int i, N = 10;
double Res = 0.0;
for (i = 0; i < N; i++) {
    Res = Res + 1.0;
}
// Или через while:
i = 0;
while (i < N) {
    Res = Res + 1.0;
    i = i + 1;
}```
**Разбор ассемблерного кода:**

| Адрес | Инструкция           | Пояснение                                                                                                                              |
| :---- | :------------------- | :------------------------------------------------------------------------------------------------------------------------------------- |
| `001` | `XOR R1, R1, R1`     | Обнуление регистра R1 (i = 0). `XOR` числа самого на себя дает 0.                                                                      |
| `004` | `ST R1, [1000]`      | Сохранение 0 в память по адресу переменной `i` ([1000]).                                                                               |
| `008` | `LD [1000], R1`      | **Начало проверки условия цикла:** Загружаем значение `i` из памяти в R1.                                                              |
| `012` | `LD [1004], R2`      | Загружаем значение `N` (10) из памяти в R2.                                                                                            |
| `016` | `CMP R1, R2`         | Сравниваем `i` (R1) и `N` (R2). Устанавливаются флаги.                                                                                |
| `020` | `JGE [+32]`          | **Проверка условия выхода:** Если R1 >= R2 (т.е. `i >= N`), перейти на адрес 020+32=052 (инструкция `NOP`), выходим из цикла.         |
| `024` | `LD [1008], F1`      | **Тело цикла:** Загружаем текущее значение `Res` в вещественный регистр F1.                                                            |
| `028` | `LD [1016], F2`      | Загружаем константу `1.0` в вещественный регистр F2.                                                                                   |
| `032` | `FADD F1, F2, F3`    | Выполняем сложение: F3 = F1 + F2.                                                                                                     |
| `036` | `ST F3, [1008]`      | Сохраняем результат сложения (F3) обратно в переменную `Res` ([1008]).                                                                 |
| `040` | `LD [1000], R1`      | Подготовка к инкременту: Загружаем текущее значение `i` в R1.                                                                          |
| `044` | `INC R1`             | Увеличиваем значение в R1 на 1 (i++).                                                                                                  |
| `048` | `ST R1, [1000]`      | Сохраняем новое значение `i` обратно в память.                                                                                         |
| `052` | `JMP [8]`            | **Безусловный переход** обратно к началу проверки условия цикла (на адрес 008).                                                        |
| `056` | `NOP`                | **Код после цикла:** Первая инструкция после завершения цикла. "No Operation" — инструкция-заглушка.                                   |

**Схема работы цикла в ассемблерном коде:**
1.  **Инициализация:** Переменная цикла инициализируется (обнуляется).
2.  **Проверка условия (метка цикла):**
    *   Значения переменной цикла и предела загружаются из памяти в регистры.
    *   Производится их сравнение (`CMP`).
    *   Условный переход (`JGE`) проверяет условие **выхода** из цикла. Если условие истинно, происходит переход **за** тело цикла.
3.  **Тело цикла:** Выполняются все инструкции, соответствующие телу цикла на C.
4.  **Изменение переменной цикла:** Значение переменной цикла обновляется (инкрементируется и сохраняется в память).
5.  **Завершение итерации:** Безусловный переход (`JMP`) возвращает управление к шагу проверки условия.


### Перевод условного оператора `if-else` на ассемблер
```c
if (a > b) {
    c = a;
} else {
    c = b;
}
```
#### Общая схема перевода:
На ассемблере условный оператор реализуется по следующему шаблону:

1.  **Проверка условия, противоположного условию в `if`**.
    *   В условии `if` проверяется `a > b`.
    *   На ассемблере мы проверяем обратное: `a <= b`.
    *   Это делается потому, что логика ассемблера строится на **пропуске** блока `if` и переходе к блоку `else`, если основное условие не выполняется.

2.  **Условный переход на начало ветки `else`**.
    *   Если обратное условие (`a <= b`) истинно, процессор переходит к инструкциям, соответствующим ветке `else`.

3.  **Код ветки `if`**.
    *   Инструкции, которые выполняются, если исходное условие (`a > b`) истинно.

4.  **Безусловный переход за пределы всего оператора `if-else`**.
    *   После выполнения ветки `if` необходимо "перепрыгнуть" через ветку `else`, чтобы не выполнить ее код. Это делается с помощью инструкции `JMP`.

5.  **Код ветки `else`**.
    *   Инструкции, которые выполняются, если исходное условие (`a > b`) ложно.

6.  **Код, следующий после оператора `if-else`**.

#### Конкретная реализация на ассемблере:
Предполагается, что переменные `a`, `b`, `c` — целочисленные (32 бита).

| Адрес | Инструкция         | Комментарий                                                                                                                              |
| :---- | :----------------- | :--------------------------------------------------------------------------------------------------------------------------------------- |
| **0** | `LD [1000], R1`    | Загружаем значение переменной `a` в регистр `R1`.                                                                                        |
| **4** | `LD [1004], R2`    | Загружаем значение переменной `b` в регистр `R2`.                                                                                        |
| **8** | `CMP R1, R2`       | **Проверка обратного условия:** Сравниваем `a` (R1) и `b` (R2). Устанавливаются флаги.                                                  |
| **12**| `JLE [+12]`        | **Условный переход на `else`:** Если `a <= b` (R1 <= R2), переходим на адрес 12+12=**24** (начало ветки `else`).                         |
| **16**| `LD [1000], R3`    | **Ветка `if` (начало):** Загружаем `a` в `R3`. <br> *Вопрос с пары: Почему не использовать `R1`? (Обсудят позже)*                       |
| **20**| `ST R3, [1008]`    | Сохраняем значение из `R3` в переменную `c` (`c = a;`).                                                                                  |
| **24**| `JMP [+8]`         | **Безусловный переход за пределы `if-else`:** Переходим на адрес 24+8=**32** (код после оператора), чтобы пропустить ветку `else`.       |
| **28**| `LD [1004], R3`    | **Ветка `else` (начало):** Загружаем `b` в `R3`.                                                                                         |
| **32**| `ST R3, [1008]`    | Сохраняем значение из `R3` в переменную `c` (`c = b;`).                                                                                  |
| **36**| `NOP`              | **Код после оператора `if-else`:** Первая инструкция после завершения всей конструкции.                                                  |

*Визуализация логики переходов:*
```
[Проверка a<=b] --(Да)--> [Else: c = b] --+
       |                                  |
      (Нет)                               |
       |                                  |
[If: c = a] --(JMP)--> [Код после if-else] <--+
```
### Организация памяти (Memory Map)

1.  **Линейное адресное пространство**: Память представляет собой массив ячеек с адресами от 0 до максимума.
2.  **Разделение на регионы**:
    *   **Регион кода (Code Section)**: Адреса **0–999**. Здесь хранятся инструкции программы.
    *   **Регион данных (Data Section)**: Адреса, начиная с **1000**. Здесь хранятся переменные и константы.
3.  **Размер инструкции**: Каждая инструкция ассемблера занимает **4 байта**. Поэтому адреса инструкций идут с шагом 4: 0, 4, 8, 12, ...
4.  **Размер переменных**:
    *   Целочисленные (`int`): 4 байта.
    *   Вещественные (`double`): 8 байт.
#### Пример карты памяти для программы с `if-else`:

| Адрес | Переменная | Размер | Пояснение                       |
| :---- | :--------- | :----- | :------------------------------ |
| 1000  | `a`        | 4 байта | Целочисленная переменная `a`.   |
| 1004  | `b`        | 4 байта | Целочисленная переменная `b`.   |
| 1008  | `c`        | 4 байта | Целочисленная переменная `c`.   |
| 1012  | ...        |        | Следующая свободная ячейка.     |

*Примечание:* Если бы в программе была переменная `double Res;`, она заняла бы адреса, например, 1012–1019 (8 байт).
### Адресация в инструкциях перехода

1.  **Абсолютные адреса**:
    *   Указывается точный адрес инструкции, на которую нужно перейти.
    *   **Используется в инструкции `JMP`**.
    *   *Пример:* `JMP [36]` — перейти на инструкцию по адресу 36.

2.  **Относительные адреса**:
    *   Указывается **смещение** (offset) относительно адреса **текущей** инструкции.
    *   **Используется в инструкциях условного перехода `JXX`**.
    *   *Как вычисляется:* `Требуемый_адрес - Текущий_адрес`.
        *   `Текущий_адрес` — это адрес инструкции, следующей за условным переходом (т.е. `IP + 4`).
    *   *Пример:* Инструкция `JLE` находится по адресу 12. Мы хотим перейти на адрес 28.
        *   Адрес следующей инструкции после `JLE` = 12 + 4 = 16.
        *   Требуемое смещение = 28 - 16 = **+12**.
        *   Инструкция будет записана как `JLE [+12]`.
    *   Положительное смещение — переход **вперед** по коду.
    *   Отрицательное смещение — переход **назад** по коду (часто используется в циклах).
### Реализация цикла `while` на ассемблере
```c
i = 0;
while (i < N) {
    Res = Res + 1.0;
    i = i + 1;
}
```

#### Общая схема перевода цикла:

1.  **Инициализация переменной цикла** (`i = 0`).
2.  **Проверка условия, противоположного условию цикла** (Проверка условия *выхода*).
    *   Условие в цикле: `i < N`.
    *   Проверяем на ассемблере обратное: `i >= N`.
3.  **Условный переход на код после цикла**. Если условие выхода (`i >= N`) истинно, происходит переход за пределы цикла.
4.  **Код тела цикла**. Инструкции, которые выполняются на каждой итерации.
5.  **Изменение переменной цикла** (`i = i + 1`).
6.  **Безусловный переход обратно к проверке условия**.
7.  **Код после завершения цикла**.

#### Конкретная реализация на ассемблере:

**Карта памяти:**
| Адрес | Переменная | Значение | Размер | Пояснение                       |
| 1000  | `i`        | (не иниц.) | 4 байта | Целочисленная переменная цикла. |
| 1004  | `N`        | `10`     | 4 байта | Целочисленный предел цикла.     |
| 1008  | `Res`      | `0.0`    | 8 байта | Вещественный результат.         |
| 1016  | `1.0`      | `1.0`    | 8 байта | Вещественная константа.         |

| Адрес  | Инструкция        | Комментарий                                                                                       |
| :----- | :---------------- | :------------------------------------------------------------------------------------------------ |
| **0**  | `XOR R1, R1, R1`  | **Инициализация:** Обнуляем регистр `R1` (i = 0). `XOR` регистра с самим собой дает 0.            |
| **4**  | `ST R1, [1000]`   | Сохраняем 0 в переменную `i` по адресу [1000].                                                    |
| **8**  | `LD [1000], R1`   | **Начало проверки условия:** Загружаем текущее значение `i` в `R1`.                               |
| **12** | `LD [1004], R2`   | Загружаем значение `N` (10) в `R2`.                                                               |

| **24** | `LD [1008], F1`   | **Тело цикла (начало):** Загружаем текущее значение `Res` в `F1`.                                 |
| **28** | `LD [1016], F2`   | Загружаем константу `1.0` в `F2`.                                                                 |

| **36** | `ST F3, [1008]`   | Сохраняем результат обратно в переменную `Res`.                                                   |

### Важные технические детали

#### Адресация в переходах

*   **`JMP` (безусловный переход)**: Использует **абсолютные адреса**. Указывается точный адрес инструкции для перехода (например, `JMP [8]`).
*   **`JXX` (условный переход)**: Использует **относительные адреса**. Указывается смещение от адреса **следующей** инструкции.
    *   *Пример:* Инструкция `JGE` находится по адресу 20. Мы хотим перейти на адрес 52.
    *   Адрес следующей инструкции после `JGE` = 20 + 4 = 24.
    *   Требуемое смещение = 52 - 24 = **+28**.
    *   Инструкция записывается как `JGE [+28]`.

*   **Как создать бесконечный цикл?** Если в условном переходе указать отрицательное смещение, например, `JMP [-4]`, то процессор будет постоянно возвращаться к одной и той же инструкции, создавая бесконечный цикл.
#### Режимы компиляции: Debug vs Release
*   **Режим Debug (Отладка)**:
    *   **Цель:** Упростить отладку, сделать поведение программы предсказуемым и независимым.
    *   **Принципы:**
        1.  **Непрерывность:** Код одной конструкции высокого уровня должен быть непрерывным блоком в ассемблере.
        2.  **Самодостаточность:** Каждый блок кода не должен зависеть от того, что было до или после него (например, значения в регистрах). Это гарантирует, что блок можно переместить, и он будет работать корректно.
        3.  **Контроль состояния памяти:** После выполнения блока состояние памяти должно точно соответствовать результату операции высокого уровня.
    *   *Пример:* В ветке `if` значение переменной `a` загружается в регистр заново, даже если оно уже было загружено ранее при проверке условия.

*   **Режим Release (Поставка)**:
    *   **Цель:** Максимальная производительность и минимальный размер кода.
    *   **Принципы:** Компилятор активно оптимизирует код, используя уже загруженные в регистры значения, переставляя инструкции и т.д. Код перестает быть "самодостаточным".

### Домашнее задание

**Задача:** Написать программу на рассмотренном псевдоассемблере, которая вычисляет **A^B**, где:
*   `A` — число типа `double`.
*   `B` — целое положительное число типа `unsigned int`.

**Требования:**
1.  Необходимо использовать **алгоритм быстрого возведения в степень**. Цель — выполнить возведение в степень (например, в 100-ю) за логарифмическое число операций (порядка 10 умножений), а не за линейное (100 умножений).
2.  Результат должен сохраняться в отдельную переменную (например, `Result`).
3.  Необходимо предварительно нарисовать **карту памяти**, указав адреса всех переменных и констант.
4.  Реализовать программу на псевдоассемблере, следуя правилам архитектуры (Load/Store, разделение целочисленных и вещественных операций, абсолютные/относительные переходы).

Отлично, давайте выполним домашнее задание и реализуем алгоритм быстрого возведения в степень.

## Алгоритм быстрого возведения в степень

**Идея алгоритма:** 
- Если степень четная: A^B = (A^(B/2))²
- Если степень нечетная: A^B = A × (A^(B-1))
- Это позволяет уменьшить количество операций с O(B) до O(log B)

## Карта памяти

| Адрес | Переменная | Значение | Размер | Описание |
|-------|------------|----------|---------|-----------|
| 1000 | `A` | (исходное) | 8 байт | Основание (double) |
| 1008 | `B` | (исходное) | 4 байта | Показатель степени (unsigned int) |
| 1012 | `Result` | 1.0 | 8 байт | Результат A^B |
| 1020 | `temp` | 1.0 | 8 байт | Временная переменная |
| 1028 | `ONE` | 1.0 | 8 байт | Константа 1.0 |
| 1036 | `ONE_INT` | 1 | 4 байта | Целочисленная константа 1 |

## Программа на псевдоассемблере

```asm
; Инициализация: Result = 1.0, temp = A, степень = B
000 XOR R1, R1, R1      ; R1 = 0
004 ST R1, [1012]       ; Обнуляем Result (младшие 32 бита)
008 ST R1, [1016]       ; Обнуляем Result (старшие 32 бита)
012 LD [1028], F1       ; F1 = 1.0
016 ST F1, [1012]       ; Result = 1.0
020 LD [1000], F2       ; F2 = A (основание)
024 ST F2, [1020]       ; temp = A
028 LD [1008], R2       ; R2 = B (степень)

; Начало основного цикла
032 LD [1008], R3       ; Загружаем B в R3 (текущая степень)
036 CMP R3, R1          ; Сравниваем степень с 0
040 JE [+64]            ; Если степень == 0, переходим к концу (032+64=096)

; Проверяем четность степени
044 LD [1036], R4       ; R4 = 1
048 AND R3, R4, R5      ; R5 = степень & 1 (проверяем младший бит)
052 CMP R5, R1          ; Сравниваем с 0
056 JE [+24]            ; Если четная, переходим к возведению в квадрат (056+24=080)

; Нечетная степень: Result = Result × temp
060 LD [1012], F3       ; F3 = Result
064 LD [1020], F4       ; F4 = temp
068 FMUL F3, F4, F5     ; F5 = Result × temp
072 ST F5, [1012]       ; Result = Result × temp

; Уменьшаем степень на 1
076 SUB R3, R4, R3      ; R3 = степень - 1
080 ST R3, [1008]       ; Сохраняем новое значение степени

; Четная степень или после нечетной: temp = temp × temp
084 LD [1020], F6       ; F6 = temp
088 FMUL F6, F6, F7     ; F7 = temp × temp
092 ST F7, [1020]       ; temp = temp²

; Делим степень на 2 (сдвиг вправо)
096 SHR R3, R4, R3      ; R3 = степень >> 1 (деление на 2)
100 ST R3, [1008]       ; Сохраняем новое значение степени

; Безусловный переход к началу цикла
104 JMP [32]            ; Возврат к проверке условия

; Конец программы
108 NOP
```

## Подробное объяснение работы алгоритма

### Принцип работы:
Алгоритм использует двоичное представление степени. Например, для вычисления A¹³:
- 13 в двоичной системе: 1101
- A¹³ = A^(8+4+1) = A⁸ × A⁴ × A¹

### Пошаговое выполнение для A¹³:

**Инициализация:**
- Result = 1.0
- temp = A
- степень = 13

**Итерация 1 (степень = 13 - нечетная):**
- Result = 1.0 × A = A
- степень = 12
- temp = A × A = A²
- степень = 12 / 2 = 6

**Итерация 2 (степень = 6 - четная):**
- temp = A² × A² = A⁴
- степень = 6 / 2 = 3

**Итерация 3 (степень = 3 - нечетная):**
- Result = A × A⁴ = A⁵
- степень = 2
- temp = A⁴ × A⁴ = A⁸
- степень = 2 / 2 = 1

**Итерация 4 (степень = 1 - нечетная):**
- Result = A⁵ × A⁸ = A¹³
- степень = 0
- Завершение цикла

### Объяснение ключевых моментов кода:

1. **Проверка четности:** `AND R3, R4, R5` - побитовое И с 1 проверяет младший бит
2. **Деление на 2:** `SHR R3, R4, R3` - сдвиг вправо на 1 эквивалентен целочисленному делению на 2
3. **Работа с вещественными числами:** Все операции с основанием и результатом используют вещественные регистры (F1-F7)
4. **Работа с целыми числами:** Операции со степенью используют целочисленные регистры (R1-R5)

### Преимущества алгоритма:
- **Сложность:** O(log B) вместо O(B)
- **Пример:** Для B = 1000 потребуется ~10 итераций вместо 1000
- **Эффективность:** Минимальное количество операций умножения

Программа корректно обрабатывает все положительные целые степени и завершается, когда степень становится равной 0.
