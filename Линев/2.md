
#### Обзор алгоритма бинарного возведения в степень
**Идея алгоритма (псевдокод):**
*   **Вход:** `x` (основание), `n` (показатель степени).
*   **Выход:** `x^n`.
*   **Инициализация:** `result = 1.0`.
```pseudocode
пока n > 0:
    если n нечётное (n % 2 != 0):
        result = result * x
        n = n - 1
    x = x * x
    n = n / 2
```

*   Основание `x` последовательно возводится в квадрат, приобретая значения `x^1, x^2, x^4, x^8, ...`.
*   Показатель степени `n` обрабатывается побитово. На каждом шаге он делится на 2 (сдвиг вправо).
*   Если младший бит `n` равен 1 (т.е. число нечётное), текущее значение `x` умножается на накапливаемый результат.
*   Вычитание единицы (`n = n - 1`) в случае нечётного `n` нужно, чтобы сделать число чётным перед делением на 2. **Ключевой момент:** лектор отметил, что эту операцию можно исключить.

#### 2. Оптимизация алгоритма: Отказ от операции деления

**Оптимизация №1: Проверка на чётность через побитовое И**
*   Вместо `(n % 2 != 0)` используется побитовая операция `(n & 1) != 0`.
*   Выражение `n & 1` возвращает значение младшего бита числа `n`. Если он равен 1, число нечётное.

**Оптимизация №2: Деление на 2 через побитовый сдвиг**
*   Вместо `n = n / 2` используется операция побитового сдвига вправо `n = n >> 1`.
*   Для целых неотрицательных чисел сдвиг вправо на 1 бит эквивалентен целочисленному делению на 2.

**Оптимизация №3: Устранение операции `n = n - 1`**
*   Лектор объяснил, что операция `n = n - 1` для нечётного `n` фактически обнуляет его младший бит.
*   Однако эта же операция выполняется *неявно* при последующем сдвиге `n >> 1`, так как младший бит просто "отбрасывается".
*   **Вывод:** Операцию `n = n - 1` можно безопасно убрать, так как необходимую коррекцию выполняет сдвиг.

```pseudocode
double A, Res = 1.0;
int B;

while (B > 0):
    if (B & 1 != 0):    // Проверка младшего бита
        Res = Res * A
    A = A * A           // Возведение в квадрат
    B = B >> 1          // Целочисленное деление на 2
```

#### План реализации на ассемблере
**А. Распределение памяти для переменных**
*   `[1000]`: Переменная `A` (тип double, занимает 8 байт).
*   `[1008]`: Переменная `Res` (результат, тип double, начальное значение `1.0`).
*   `[1016]`: Переменная `B` (показатель степени, тип int, занимает 4 байта).
*   `[1020]`: Константа `1` (целочисленная, для использования в операции `AND`).

**Б. Распределение регистров**
*   `F1`: Хранит значение переменной `A`.
*   `F2`: Хранит значение переменной `Res`.
*   `R1`: Хранит значение переменной `B`.
*   `R2`: Хранит константу `1`.
*   `R3`, `R4`: Используются как временные регистры для промежуточных вычислений (например, для хранения нуля и результата операции `AND`).

#### Построчный разбор ассемблерного кода
1.  **`00 LD [1016], R1`**
    *   Начало цикла `while`. Загружаем значение переменной `B` из памяти (адрес `[1016]`) в регистр `R1`.

2.  **`04 XOR R3, R3, R3`**
    *   Получить значение `0` для сравнения. Операция `XOR` регистра с самим собой обнуляет его. Регистр `R3` теперь содержит `0`.

3.  **`08 CMP R1, R3`**
    *   Сравнить `B` (в `R1`) с `0` (в `R3`). Это проверка условия продолжения цикла `while (B > 0)`.

4.  **`12 JLE [+72]`**
    *   Условный переход. Если условие `B <= 0` (`JLE` - Jump if Less or Equal) выполняется, происходит выход из цикла. Смещение `[+72]` условное и указывает на адрес за пределами цикла (в данном случае, на инструкцию `NOP` по адресу `88`).

5.  **`16 LD [1016], R1`**
    *   Начало условного оператора `if`. Снова загружаем `B` в `R1` (возможно, его значение могло измениться, но на данном этапе это подготовка к проверке бита).

6.  **`20 LD [1020], R2`**
    *   Загружаем целочисленную константу `1` из адреса `[1020]` в регистр `R2`.

7.  **`24 AND R1, R2, R4`**
    *   Выполняем побитовое `AND` между `B` (`R1`) и `1` (`R2`). Результат (значение младшего бита `B`) сохраняется в регистр `R4`.

8.  **`28 XOR R3, R3, R3`**
    *  Повторно обнуляем регистр `R3` (хотя он уже был обнулён, это делается для ясности или если его значение могло быть изменено).

9.  **`32 CMP R4, R3`**
    *  Сравниваем результат операции `AND` (в `R4`) с `0` (в `R3`). Это проверка условия `if (B & 1 != 0)`.

10. **`36 JE [+16]`**
    *   Условный переход. Если результат сравнения равен нулю (`JE` - Jump if Equal), значит, младший бит `B` равен `0` (число чётное), и выполнение тела оператора `if` нужно пропустить. Переход происходит на адрес `52` (к операции `A = A * A`).

11. **`40 LD [1008], F2`** ... **`52 ST F2, [1008]`**
    *  **Тело оператора `if`.** Если условие выполнилось (число `B` нечётное), вычисляется `Res = Res * A`.
    *   `40 LD [1008], F2`: Загружаем текущее значение `Res` в регистр `F2`.
    *   `44 LD [1000], F1`: Загружаем значение `A` в регистр `F1`.
    *   `48 FMUL F2, F1, F2`: Умножаем `Res (F2)` на `A (F1)`. Результат сохраняется обратно в `F2`.
    *   `52 ST F2, [1008]`: Сохраняем новое значение `Res` из регистра `F2` обратно в память по адресу `[1008]`.

12. **`56 LD [1000], F1`** ... **`64 ST F1, [1000]`**
    *   **Код после `if`.** Вычисление `A = A * A`.
    *   `56 LD [1000], F1`: Загружаем значение `A` в регистр `F1`.
    *   `60 FMUL F1, F1, F1`: Умножаем `A (F1)` на самого себя (возводим в квадрат).
    *   `64 ST F1, [1000]`: Сохраняем новое значение `A` обратно в память по адресу `[1000]`.

13. **`68 LD [1016], R1`** ... **`80 ST R1, [1016]`**
    *   Вычисление `B = B >> 1`.
    *   `68 LD [1016], R1`: Загружаем значение `B` в регистр `R1`.
    *   `72 LD [1020], R2`: Загружаем константу `1` в регистр `R2` (в данном контексте она используется как количество бит для сдвига).
    *   `76 SHR R1, R2, R1`: Выполняем арифметический или логический сдвиг вправо (SHR) значения в `R1` на количество бит, указанное в `R2` (т.е. на 1). Результат помещается в `R1`.
    *   `80 ST R1, [1016]`: Сохраняем новое значение `B` обратно в память.

14.  **`84 JMP [00]`**
    *   Конец тела цикла. Эта инструкция реализует безусловный переход (`JMP`) обратно к началу цикла, на адрес `00`, где находится первая инструкция (`LD [1016], R1`) для проверки условия `while (B > 0)`.

15.  **`88 NOP`**
    *   **Метка выхода из цикла. Сюда происходит переход по условию `JLE [+72]`, когда `B` становится меньше или равен нулю. `NOP` (No Operation) — инструкция-заглушка, не выполняющая никаких действий, отмечающая конец алгоритма.

**Расчёт смещений для переходов:**

*   **`12 JLE [+72]`**: Текущий адрес инструкции — `12`. Адрес следующей инструкции — `16`. Нужно перейти на адрес `88`.
    *   Смещение = `88 - 16 = 72`.
*   **`36 JE [+16]`**: Текущий адрес инструкции — `36`. Адрес следующей инструкции — `40`. Нужно перейти на адрес `56` (начало блока `A = A * A`, чтобы пропустить тело `if`).
    *   Смещение = `56 - 40 = 16`.
*   **`84 JMP [00]`**: Это безусловный переход на абсолютный адрес `00`.

####  Способы формирования адресов в памяти
**Общая модель памяти программы:**

1.  **Регион кода (Code Region):**
    *   Располагается в младших адресах (начиная с `0`).
    *   Содержит инструкции программы.
    *   **Важно:** Адреса инструкций известны на этапе компиляции/линковки и не меняются от запуска к запуску.

2.  **Регион данных (Data Region):**
    *   Располагается сразу за кодом (в примере — начиная с адреса `1000`).
    *   Содержит **глобальные и статические переменные**, а также **значения констант**.
    *   Адреса этих переменных также фиксированы и известны до запуска программы.

3.  **Куча (Heap):**
    *   Область для **динамически распределяемой памяти**.
    *   Используется при вызове операторов `new`, `malloc` и т.д.
    *   Растёт в сторону увеличения адресов.

4.  **Стек (Stack):**
    *   Область для хранения локальных переменных функций, параметров и данных о вызовах.
    *   Обычно располагается в старших адресах и растёт в сторону их уменьшения.

#### Способы адресации

**Способ 1: Прямая (Абсолютная) Адресация**

*   Адрес в инструкции задаётся в виде **непосредственного числового значения (константы)**.
*  Для доступа к данным с **фиксированными адресами**. В первую очередь, к глобальным и статическим переменным в регионе данных.
*   **Пример:**
    *   Объявлены глобальные переменные: `int a` (по адресу `1000`), `int i` (по адресу `1004`).
    *   Инструкция для доступа к `i`: `LD [1004], R1`. Адрес `1004` жёстко "вшит" в код.

**Способ 2: Косвенная (Регистровая) Адресация**

*   Адрес в памяти, к которому нужно обратиться, **хранится в одном из регистров** процессора.
*   Для работы с указателями и динамической памятью в **куче**.
*   **Пример:**
    *   Объявлен указатель: `int *pi` (размер — 4 байта, размещён по адресу `1008` в регионе данных).
    *   В коде выполняется операция: `pi = new int;`
        1.  Оператор `new` выделяет блок памяти в **куче** и возвращает его адрес.
        2.  Этот адрес сохраняется в переменной-указателе `pi` (т.е. по адресу `1008` записывается число — адрес в куче).
    *   Чтобы получить значение, на которое указывает `pi`, процессор должен:
        1.  Загрузить значение из переменной `pi` (адрес `1008`) в регистр, например, `R3`. Теперь в `R3` лежит адрес в куче.
        2.  Выполнить инструкцию косвенной загрузки: `LD [R3], R4`. Это означает: "взять адрес из регистра `R3`, обратиться по этому адресу в память и загрузить значение в регистр `R4`".

**Способ 3: Адресация «Регистр + Константа»**

**Подвариант 3.1: Константа — базовый адрес, Регистр — смещение**

*   В инструкции жёстко задан базовый адрес (например, начало массива), а смещение относительно этого адреса вычисляется и хранится в регистре.
*   **Пример:** Доступ к элементу массива по индексу `i`.
    *   Объявлен массив `int B[10]`. Начало массива (`B[0]`) размещено по адресу `1012`.
    *   Код: `a = B[i];`
    *   **Реализация:**
        1.  `LD [1004], R1` // Загружаем значение переменной `i` в регистр `R1`.
        2.  `LD [1052], R3` // Загружаем константу `4` (размер элемента `int`) в регистр `R3`. Адрес `1052` специально выделен в памяти для хранения этой константы.
        3.  `MUL R1, R3, R4` // Умножаем `i (R1)` на размер элемента `(R3)`. Результат — смещение в байтах —保存在 в регистр `R4`.
        4.  `LD [1012 + R4], R5` // Загружаем элемент `B[i]` по адресу `1012` (база) + `R4` (смещение). Результат в `R5`.

**Подвариант 3.2: Константа — смещение, Регистр — базовый адрес**

*   В инструкции жёстко задано смещение, а базовый адрес (например, указатель на массив) хранится в регистре.
*   **Пример:** Доступ к элементу массива через указатель.
    *   Объявлен указатель: `int *pi = new int[10];`
    *   Код: `a = pi[5];` (эквивалентно `a = *(pi + 5)`)
    *   **Реализация:**
        1.  `LD [1008], R2` // Загружаем значение указателя `pi` (адрес начала массива в куче) в регистр `R2`.
        2.  `LD [R2 + 20], R1` // Загружаем значение 5-го элемента. Смещение `20` вычисляется на этапе компиляции: `5 (индекс) * 4 (размер int) = 20`.

**Способ 4: Адресация «Регистр + Регистр»**

*  И базовый адрес, и смещение хранятся в регистрах. Это самый гибкий, но и самый ресурсоёмкий способ.
*   **Пример:** Доступ к элементу массива через указатель и переменный индекс.
    *   Код: `a = pi[i];`
    *   **Реализация:** Комбинация подвариантов 3.1 и 3.2.
        1.  Загрузить указатель `pi` в регистр (база).
        2.  Загрузить индекс `i` в другой регистр.
        3.  Вычислить смещение, умножив индекс на размер элемента (требует отдельной инструкции и хранения константы размера).
        4.  Выполнить загрузку по адресу `[БазовыйРегистр + РегистрСмещение]`.

#### Практическая реализация: Сумма элементов массива
**Постановка задачи:**
Вычислить сумму элементов вещественного массива `A` из 10 элементов.
```c
double A[10];
double res = 0.0;
int i;
int n = 10;
for (i = 0; i < n; i++) {
    res = res + A[i];
}
```
**План памяти (Карта памяти):**

*   `[1000]`: `res` (double, результат, начальное значение 0.0)
*   `[1008]`: `A[0]` (double, первый элемент массива)
*   `[1016]`: `A[1]`
*   `...`
*   `[1080]`: `A[9]` (последний элемент; `1008 + 9*8 = 1080`)
*   `[1088]`: `i` (int, счётчик цикла)
*   `[1092]`: `n` (int, размер массива, значение 10)
*   `[1096]`: `size_of_element` (int, константа = 8, размер элемента `double`)

**Распределение регистров:**

*   `F1`: Хранит текущее значение `res`.
*   `F2`: Хранит текущий элемент массива `A[i]` для сложения.
*   `R1`: Хранит значение переменной `i`.
*   `R2`: Хранит значение переменной `n`.
*   `R3`: Хранит константу размера элемента массива (8).

**Начало реализации на ассемблере:**

1.  **Инициализация `i = 0`:**
    *   `00: XOR R1, R1, R1` // Обнуляем регистр `R1` с помощью XOR.
    * `04: ST R1, [1088]` // Сохраняем 0 из `R1` в память по адресу переменной `i` (`[1088]`).

2.  **Начало цикла `while (i < n)`:**
    *   `08: LD [1088], R1` // Загружаем текущее значение `i` в `R1`.
    *   `12: LD [1092], R2` // Загружаем значение `n` в `R2`.
    *   `16: CMP R1, R2` // Сравниваем `i` (R1) и `n` (R2).
    *  `20: JGE [+??]` // Условный переход *за пределы цикла*. Если `i >= n` (противоположное условие `i < n`), выходим из цикла. Точное смещение `[+??]` будет вычислено позже, после написания всего тела цикла.

3.  **Тело цикла: `res = res + A[i]`**
    *   **Загрузка `res`:**
        *   `24: LD [1000], F1` // Загружаем текущее значение `res` в регистр `F1`.
    *   **Вычисление адреса `A[i]` и загрузка элемента:**
        *   `28: LD [1088], R1` // Загружаем `i` в `R1` (если значение было потеряно).
        *   `32: LD [1096], R3` // Загружаем константу `8` (размер `double`) в `R3`.
        *   `36: MUL R1, R3, R4` // Вычисляем смещение: `i * 8`. Результат в `R4`.
        *   `40: LD [1008 + R4], F2` // Загружаем элемент `A[i]` в `F2`. Адрес вычисляется как базовый адрес массива `1008` плюс смещение `R4`.
    *   **Сложение и сохранение результата:**
        *   `44: FADD F1, F2, F1` // Складываем `res (F1)` и `A[i] (F2)`. Результат сохраняем обратно в `F1`.
        *   `48: ST F1, [1000]` // Сохраняем новое значение `res` из `F1` обратно в память.

4.  **Инкремент счётчика `i++`:**
    *   `52: LD [1088], R1` // Загружаем текущее значение `i` в `R1`.
    *   `56: INC R1` // Увеличиваем значение в `R1` на 1.
    *   `60: ST R1, [1088]` // Сохраняем обновлённое значение `i` обратно в память.

5.  **Завершение цикла:**
    *   `64: JMP [08]` // Безусловный переход обратно к началу цикла (на адрес `08`) для проверки условия.

Лектор отмечает, что для загрузки одного элемента массива потребовалось **5 инструкций**, а на всё тело цикла, включая инкремент счётчика, — **7 инструкций**.

#### Завершение реализации и расчёт адресов переходов

**Завершение цикла:**

6.  **`64: JMP [08]`**
    *   **Цель:** Безусловный переход обратно к началу цикла, на инструкцию `LD [1088], R1` по адресу `08`, для проверки условия `i < n`.

7.  **`68: NOP`**
    *   **Цель:** Метка выхода из цикла. Сюда должен вести условный переход `JGE` при завершении цикла.

**Расчёт смещений для переходов:**

*   **Адрес 1 (`20: JGE [+??]`):**
    *   Текущий адрес инструкции: `20`.
    *   Адрес следующей инструкции: `24`.
    *   Целевой адрес перехода (конец цикла): `68`.
    *   **Смещение = `68 - 24 = 44`.** Инструкция принимает вид: `20: JGE [+44]`.

*   **Адрес 2 (`64: JMP [08]`):**
    *   **Цель:** Безусловный переход на начало цикла.
    *   Используется абсолютный адрес. Инструкция: `64: JMP [08]`.

На этом код "отладочной" (debug) версии программы завершён.

#### Оптимизированная (Release) версия программы

**Идея оптимизации:**
*   **Изменение порядка обхода:** Цикл проходит массив не от первого к последнему элементу, а **от последнего к первому**.
*   **Совмещение операций:** Операция изменения индекса (`i`) и проверки условия цикла совмещены в одной арифметической команде.
*   **Минимизация обращений к памяти:** Переменные максимально долго хранятся в регистрах, избегая лишних операций сохранения (`ST`) и загрузки (`LD`).

**Ключевой приём:**
*   Вместо отдельной инструкции `CMP` для проверки условия используется инструкция `SUB` (вычитание).
*   `SUB R4, R3, R4` не только вычисляет новое значение, но и **устанавливает флаги процессора** (нулевой флаг, флаг знака и т.д.), так же как это делает `CMP`.
*   Последующая инструкция `JNE` (Jump if Not Equal) анализирует эти флаги для принятия решения о переходе.

**Преимущества Release-версии:**
1.  **Меньше инструкций:** Отсутствуют лишние операции загрузки и сохранения в пределах цикла.
2.  **Более "умное" использование регистров:** Данные переиспользуются.
3.  **Более эффективный цикл:** Проверка условия и изменение индекса объединены.

**Недостаток Release-версии:**
*   **Сложность отладки:** Связь между строками исходного высокоуровневого кода и инструкциями ассемблера неочевидна. Инструкции могут быть переставлены местами для оптимизации. Такой код значительно сложнее отлаживать.

**Замечание по итоговому значению переменной `i`:**
*   В Debug-версии значение `i` в памяти после цикла равно 10.
*   В Release-версии лектор изначально не сохранил итоговое значение `i` в память (так как оно не используется в дальнейших вычислениях — это допустимо для оптимизации).
*   Однако, для строгого соответствия поведению Debug-версии, это значение должно быть обновлено. Лектор предлагает два способа:
    1.  `LD [1092], R1` затем `ST R1, [1088]` — загрузить `n` и сохранить в `i`. Просто, но требует обращения к памяти.
    2.  `INC R3` два раза — так как в `R3` уже хранится число `8`, два инкремента дадут `10`, которое затем можно сохранить в `i`. Этот способ быстрее, так как не требует чтения из памяти.

2.  **Дополнительное задание (Сравнение Debug и Release):**
    *   Для рассмотренных на лекции примеров программы суммирования массива (и для своей программы поиска максимума/минимума) **посчитать количество инструкций**, которые будут выполнены в Debug и Release версиях.
    *   **Цель:** Оценить, во сколько раз одна версия может быть эффективнее другой по количеству выполняемых команд, и понять, за что "борются" оптимизирующие компиляторы.

**Дополнительные пояснения от лектора:**

*   **Производительность Release:** Преимущество Release-версии заключается не только в меньшем количестве инструкций, но и в их **порядке**, который может быть более оптимальным для конвейера процессора (эта тема будет рассмотрена позже).
*   **Адреса в коде (0, 4, 8...):** Это адреса инструкций в регионе кода оперативной памяти. В данной учебной архитектуре каждая инструкция занимает 4 байта, поэтому адреса идут с шагом 4.
