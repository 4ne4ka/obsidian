## **Программа 1: Инициализация единичной матрицы 4×4**

### **Карта памяти (для обеих версий):**

| Адрес | Переменная | Значение | Размер | Описание |
|-------|------------|----------|---------|-----------|
| 1000 | `matrix[0][0]` | 0.0 | 8 байт | Элемент матрицы |
| 1008 | `matrix[0][1]` | 0.0 | 8 байт | ... |
| ... | ... | ... | ... | ... |
| 1128 | `matrix[3][3]` | 0.0 | 8 байт | Всего 16 элементов × 8 байт = 128 байт |
| 1128 | `ONE` | 1.0 | 8 байт | Константа 1.0 |
| 1136 | `ZERO` | 0.0 | 8 байт | Константа 0.0 |
| 1144 | `i` | 0 | 4 байта | Счетчик строк |
| 1148 | `j` | 0 | 4 байта | Счетчик столбцов |
| 1152 | `SIZE` | 4 | 4 байта | Размер матрицы |

### **Release версия (оптимизированная):**

```asm
; Инициализация единичной матрицы 4×4 (Release версия)

; Инициализация регистров
000: LD [1128], F1        ; F1 = 1.0
004: LD [1136], F2        ; F2 = 0.0
008: XOR R1, R1, R1       ; R1 = 0 (i)
00C: LD [1152], R2        ; R2 = 4 (SIZE)
010: LD [1156], R3        ; R3 = 8 (размер double)

; Внешний цикл (i)
014: CMP R1, R2           ; i < 4?
018: JGE [+64]            ; Выход при i >= 4 (к адресу 07C)

01C: XOR R4, R4, R4       ; R4 = 0 (j)
020: MUL R1, R2, R5       ; R5 = i * 4

; Внутренний цикл (j)
024: CMP R4, R2           ; j < 4?
028: JGE [+40]            ; Следующая строка при j >= 4 (к адресу 068)

02C: ADD R5, R4, R6       ; R6 = i*4 + j
030: MUL R6, R3, R6       ; R6 = (i*4 + j) * 8
034: ADD R6, R7, R6       ; R6 = 1000 + смещение (R7 = 1000)

; Быстрая проверка i == j
038: CMP R1, R4           ; i == j?
03C: JNE [+8]             ; Если не равно, к адресу 044

040: ST F1, [R6]          ; matrix[i][j] = 1.0
044: JMP [+4]             ; К адресу 048

048: ST F2, [R6]          ; matrix[i][j] = 0.0

04C: INC R4               ; j++
050: JMP [024]            ; Возврат к проверке внутреннего цикла

; Следующая строка
054: NOP                  ; Метка конца внутреннего цикла
058: INC R1               ; i++
05C: JMP [014]            ; Возврат к проверке внешнего цикла

; Конец программы
060: NOP
```


## **Программа 2: Проверка симметричности массива из 10 вещественных элементов**

*Симметричный массив: `array[i] == array[9-i]` для всех i от 0 до 4*

### **Карта памяти (для обеих версий):**

| Адрес | Переменная | Значение | Размер | Описание |
|-------|------------|----------|---------|-----------|
| 2000 | `array[0]` | (значение) | 8 байт | Элемент массива |
| 2008 | `array[1]` | (значение) | 8 байт | ... |
| ... | ... | ... | ... | ... |
| 2072 | `array[9]` | (значение) | 8 байт | 10 элементов × 8 байт = 80 байт |
| 2080 | `i` | 0 | 4 байта | Счетчик |
| 2084 | `size` | 10 | 4 байта | Размер массива |
| 2088 | `half` | 5 | 4 байта | Половина размера |
| 2092 | `is_symmetric` | 1 | 4 байта | Флаг результата (1 = симметричный) |

### **Release версия:**

```asm
; Проверка симметричности массива (Release версия)

; Инициализация регистров
000: XOR R1, R1, R1       ; R1 = 0 (i)
004: LD R2, [2092]        ; R2 = 1
008: ST R2, [2092]        ; is_symmetric = 1
00C: LD R3, [2088]        ; R3 = half (5)
010: LD R4, [2100]        ; R4 = 8 (размер double)
014: LD R5, [2084]        ; R5 = size (10)
018: DEC R5               ; R5 = 9
01C: MOV R6, 2000         ; R6 = базовый адрес массива

; Основной цикл
020: CMP R1, R3           ; i < 5?
024: JGE [+48]            ; Выход при i >= 5 (к адресу 06C)

; Загрузка array[i]
028: MUL R1, R4, R7       ; R7 = i * 8
02C: ADD R6, R7, R7       ; R7 = 2000 + i*8
030: LD [R7], F1          ; F1 = array[i]

; Загрузка array[9-i]
034: SUB R5, R1, R8       ; R8 = 9 - i
038: MUL R8, R4, R8       ; R8 = (9-i) * 8
03C: ADD R6, R8, R8       ; R8 = 2000 + (9-i)*8
040: LD [R8], F2          ; F2 = array[9-i]

; Сравнение и установка флага
044: FCMP F1, F2          ; Сравниваем
048: JE [+8]              ; Если равны, пропускаем (к адресу 050)
04C: XOR R2, R2, R2       ; R2 = 0
050: ST R2, [2092]        ; is_symmetric = 0 (если были неравные)

054: INC R1               ; i++
058: JMP [020]            ; Возврат к проверке условия

; Конец программы
05C: NOP
```


## **Программа 3: Сумма ряда Σ(1/i!) от i=0 до n**

**Математическая формула:**  
```
S = Σ(1/i!) = 1/0! + 1/1! + 1/2! + ... + 1/n!
```
Где `0! = 1` по определению.

**Рекуррентное соотношение для оптимизации:**  
```
term₀ = 1
termᵢ = termᵢ₋₁ / i
S = Σ termᵢ
```

### **Карта памяти (для обеих версий):**

| Адрес | Переменная | Значение | Размер | Описание |
|-------|------------|----------|---------|-----------|
| 3000 | `n` | (входное) | 4 байта | Верхний предел суммирования |
| 3004 | `i` | 0 | 4 байта | Счетчик |
| 3008 | `sum` | 0.0 | 8 байт | Результат суммы |
| 3016 | `term` | 1.0 | 8 байт | Текущий член ряда (1/i!) |
| 3024 | `ONE` | 1.0 | 8 байт | Константа 1.0 |
| 3032 | `ONE_INT` | 1 | 4 байта | Целочисленная константа 1 |

### **Release версия (оптимизированная):**

```asm
; Сумма ряда Σ(1/i!) от i=0 до n (Release версия)

; Инициализация регистров
000: LD [3024], F1        ; F1 = 1.0 (term)
004: MOV F2, F1           ; F2 = 1.0 (sum)
008: LD [3000], R1        ; R1 = n
00C: XOR R2, R2, R2       ; R2 = 0 (i)
010: INC R2               ; R2 = 1 (начинаем с i=1)
014: LD [3032], R3        ; R3 = 1 (константа)

; Основной цикл
018: CMP R2, R1           ; i <= n?
01C: JG [+40]             ; Выход при i > n (к адресу 05C)

; Вычисление term = term / i
020: CVTIF R2, F3         ; F3 = (double)i
024: FDIV F1, F3, F1      ; F1 = term / i

; Сложение sum = sum + term
028: FADD F2, F1, F2      ; F2 = sum + term

; Инкремент i
02C: ADD R2, R3, R2       ; R2 = i + 1

; Возврат к началу цикла
030: JMP [018]            ; Возврат к проверке условия

; Сохранение результатов
034: ST F1, [3016]        ; term (последнее значение)
038: ST F2, [3008]        ; sum
03C: ST R2, [3004]        ; i (последнее значение)

; Конец программы
040: NOP
```

## **Программа 4: Сумма ряда Σ(1/2^i) от i=0 до n**

**Математическая формула (геометрическая прогрессия):**  
```
S = Σ(1/2^i) = 1 + 1/2 + 1/4 + ... + 1/2^n
S = (1 - (1/2)^(n+1)) / (1 - 1/2) = 2 * (1 - 2^-(n+1))
```
Но для демонстрации циклического вычисления реализуем итеративно.

**Рекуррентное соотношение:**  
```
term₀ = 1
termᵢ = termᵢ₋₁ / 2
S = Σ termᵢ
```

### **Карта памяти (для обеих версий):**

| Адрес | Переменная | Значение | Размер | Описание |
|-------|------------|----------|---------|-----------|
| 4000 | `n` | (входное) | 4 байта | Верхний предел суммирования |
| 4004 | `i` | 0 | 4 байта | Счетчик |
| 4008 | `sum` | 0.0 | 8 байт | Результат суммы |
| 4016 | `term` | 1.0 | 8 байт | Текущий член ряда (1/2^i) |
| 4024 | `TWO` | 2.0 | 8 байт | Константа 2.0 |
| 4032 | `ONE` | 1.0 | 8 байт | Константа 1.0 |
| 4040 | `HALF` | 0.5 | 8 байт | Константа 0.5 |
| 4048 | `ONE_INT` | 1 | 4 байта | Целочисленная константа 1 |

### **Release версия (оптимизированная):**

```asm
; Сумма ряда Σ(1/2^i) от i=0 до n (Release версия)

; Инициализация регистров
000: LD [4032], F1        ; F1 = 1.0 (term)
004: MOV F2, F1           ; F2 = 1.0 (sum)
008: LD [4000], R1        ; R1 = n
00C: XOR R2, R2, R2       ; R2 = 0 (i)
010: LD [4040], F3        ; F3 = 0.5
014: LD [4048], R3        ; R3 = 1 (константа)

; Основной цикл
018: CMP R2, R1           ; i <= n?
01C: JG [+32]             ; Выход при i > n (к адресу 04C)

; Пропускаем вычисления для i=0
020: CMP R2, 0            ; i == 0?
024: JE [+12]             ; Если i=0, пропускаем (к адресу 030)

; Вычисление term = term / 2
028: FMUL F1, F3, F1      ; F1 = term * 0.5

; Сложение sum = sum + term
02C: FADD F2, F1, F2      ; F2 = sum + term

; Инкремент i
030: ADD R2, R3, R2       ; R2 = i + 1

; Возврат к началу цикла
034: JMP [018]            ; Возврат к проверке условия

; Сохранение результатов
038: ST F1, [4016]        ; term (последнее значение)
03C: ST F2, [4008]        ; sum
040: ST R2, [4004]        ; i (последнее значение)

; Конец программы
044: NOP
```

### **Release версия (с использованием формулы геометрической прогрессии):**

```asm
; Сумма ряда Σ(1/2^i) от i=0 до n (Release версия, формула)

; Используем формулу: S = 2 * (1 - 2^-(n+1))
000: LD [4000], R1        ; R1 = n
004: LD [4048], R2        ; R2 = 1
008: ADD R1, R2, R1       ; R1 = n + 1
00C: NEG R1, R3           ; R3 = -(n+1)

; Вычисление 2^-(n+1)
010: CVTIF R3, F1         ; F1 = -(n+1) как double
014: LD [4024], F2        ; F2 = 2.0
018: POW F2, F1, F3       ; F3 = 2^-(n+1) (предполагаем наличие инструкции POW)

; Вычисление 1 - 2^-(n+1)
01C: LD [4032], F4        ; F4 = 1.0
020: FSUB F4, F3, F5      ; F5 = 1 - 2^-(n+1)

; Умножение на 2
024: FMUL F5, F2, F6      ; F6 = 2 * (1 - 2^-(n+1))

; Сохранение результата
028: ST F6, [4008]        ; sum = результат

; Конец программы
02C: NOP
```

## **Программа 5: Найти максимум и минимум в массиве вещественных чисел**

### **Карта памяти (для обеих версий):**

| Адрес | Переменная | Значение | Размер | Описание |
|-------|------------|----------|---------|-----------|
| 5000 | `array[0]` | (значение) | 8 байт | Элемент массива |
| 5008 | `array[1]` | (значение) | 8 байт | ... |
| ... | ... | ... | ... | ... |
| 5080 | `array[15]` | (значение) | 8 байт | 16 элементов × 8 байт = 128 байт |
| 5088 | `n` | 16 | 4 байта | Размер массива |
| 5092 | `i` | 0 | 4 байта | Счетчик |
| 5096 | `max` | -∞ | 8 байт | Максимальное значение |
| 5104 | `min` | +∞ | 8 байт | Минимальное значение |
| 5112 | `NEG_INF` | -1e308 | 8 байт | Константа -∞ |

```asm
; Поиск максимума и минимума в массиве (Release версия)

; Инициализация регистров
000: LD [5112], F1        ; F1 = -∞ (max)
004: LD [5120], F2        ; F2 = +∞ (min)
008: LD [5088], R1        ; R1 = n (16)
00C: XOR R2, R2, R2       ; R2 = 0 (i)
010: LD [5136], R3        ; R3 = 8 (размер double)
014: MOV R4, 5000         ; R4 = базовый адрес массива
018: LD [5128], R5        ; R5 = 1 (константа)

; Основной цикл
01C: CMP R2, R1           ; i < n?
020: JGE [+64]            ; Выход при i >= n (к адресу 084)

; Загрузка array[i]
024: MUL R2, R3, R6       ; R6 = i * 8
028: ADD R4, R6, R6       ; R6 = 5000 + i*8
02C: LD [R6], F3          ; F3 = array[i]

; Проверка и обновление максимума
030: FCMP F3, F1          ; array[i] > max?
034: JLE [+8]             ; Если не больше, пропускаем (к адресу 03C)
038: MOV F1, F3           ; max = array[i]

; Проверка и обновление минимума
03C: FCMP F3, F2          ; array[i] < min?
040: JGE [+8]             ; Если не меньше, пропускаем (к адресу 048)
044: MOV F2, F3           ; min = array[i]

; Инкремент i
048: ADD R2, R5, R2       ; i++

; Возврат к началу цикла
04C: JMP [01C]            ; Возврат к проверке условия

; Сохранение результатов
050: ST F1, [5096]        ; max
054: ST F2, [5104]        ; min
058: ST R2, [5092]        ; i (последнее значение)

; Конец программы
05C: NOP
```


## **Программа 6: Найти максимум/минимум по абсолютному значению (без использования функции модуля)**

**Подход без использования модуля:**  
Для вещественных чисел: |x| = x, если x ≥ 0, иначе |x| = -x

### **Карта памяти (для обеих версий):**

| Адрес | Переменная | Значение | Размер | Описание |
|-------|------------|----------|---------|-----------|
| 6000 | `array[0]` | (значение) | 8 байт | Элемент массива |
| 6008 | `array[1]` | (значение) | 8 байт | ... |
| ... | ... | ... | ... | ... |
| 6080 | `array[15]` | (значение) | 8 байт | 16 элементов |
| 6088 | `n` | 16 | 4 байта | Размер массива |
| 6092 | `i` | 0 | 4 байта | Счетчик |
| 6096 | `max_abs` | 0.0 | 8 байт | Максимальное абсолютное значение |
| 6104 | `min_abs` | +∞ | 8 байт | Минимальное абсолютное значение |
| 6112 | `ZERO` | 0.0 | 8 байт | Константа 0.0 |
| 6120 | `POS_INF` | 1e308 | 8 байт | Константа +∞ |
| 6128 | `NEG_ONE` | -1.0 | 8 байт | Константа -1.0 |
| 6136 | `ONE` | 1 | 4 байта | Целочисленная константа 1 |

### **Release версия (оптимизированная):**

```asm
; Поиск максимума/минимума по абсолютному значению (Release версия)

; Инициализация регистров
000: LD [6112], F1        ; F1 = 0.0 (max_abs)
004: LD [6120], F2        ; F2 = +∞ (min_abs)
008: LD [6088], R1        ; R1 = n (16)
00C: XOR R2, R2, R2       ; R2 = 0 (i)
010: LD [6144], R3        ; R3 = 8 (размер double)
014: MOV R4, 6000         ; R4 = базовый адрес массива
018: LD [6128], F10       ; F10 = -1.0
01C: LD [6136], R5        ; R5 = 1 (константа)
020: LD [6112], F11       ; F11 = 0.0 (для сравнения)

; Основной цикл
024: CMP R2, R1           ; i < n?
028: JGE [+80]            ; Выход при i >= n (к адресу 0A8)

; Загрузка array[i]
02C: MUL R2, R3, R6       ; R6 = i * 8
030: ADD R4, R6, R6       ; R6 = 6000 + i*8
034: LD [R6], F3          ; F3 = array[i]

; Вычисление абсолютного значения
038: FCMP F3, F11         ; array[i] >= 0?
03C: JGE [+8]             ; Если >= 0, к адресу 044
040: FMUL F3, F10, F4     ; F4 = -array[i] (для отрицательных)
044: MOV F4, F3           ; F4 = array[i] (для неотрицательных)

; Теперь F4 содержит abs_value

; Проверка и обновление max_abs
048: FCMP F4, F1          ; abs_value > max_abs?
04C: JLE [+8]             ; Если не больше, пропускаем (к адресу 054)
050: MOV F1, F4           ; max_abs = abs_value

; Проверка и обновление min_abs
054: FCMP F4, F2          ; abs_value < min_abs?
058: JGE [+8]             ; Если не меньше, пропускаем (к адресу 060)
05C: MOV F2, F4           ; min_abs = abs_value

; Инкремент i
060: ADD R2, R5, R2       ; i++

; Возврат к началу цикла
064: JMP [024]            ; Возврат к проверке условия

; Сохранение результатов
068: ST F1, [6096]        ; max_abs
06C: ST F2, [6104]        ; min_abs
070: ST R2, [6092]        ; i (последнее значение)

; Конец программы
074: NOP
```

### **Альтернативная Release версия (только максимум по абсолютному значению, более оптимизированная):**

```asm
; Поиск только максимума по абсолютному значению (самая оптимизированная версия)

; Инициализация
000: LD [6112], F1        ; F1 = 0.0 (max_abs)
004: LD [6088], R1        ; R1 = n (16)

00C: MOV R3, 6000         ; R3 = базовый адрес массива

044: INC R2               ; i++

; Сохранение результата

; Конец программы

```
